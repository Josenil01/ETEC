1. Da Planta para a Fundação

Até agora, operamos no Modelo Conceitual (DER). Ele é o rascunho, focado em entender o problema do cliente ("Um Médico atende um Paciente"). Mas o SGBD (PostgreSQL, MySQL) não entende retângulos e losangos.

É hora de traduzir o desenho para o Modelo Lógico. Aqui, as ideias transformam-se em Tabelas com linhas rígidas de cruzamento de dados. Não escrevemos código SQL ainda, mas preparamos a planta exata de como a base será construída.

2. O Segredo das Chaves (Relacionamentos)

No diagrama conceitual, os relacionamentos são desenhados com linhas e losangos. Nas tabelas, não existem linhas mágicas. A ligação entre tabelas é feita através da cópia de uma Chave Primária (PK) para outra tabela, onde ela passa a chamar-se Chave Estrangeira (FK - Foreign Key).

Regra de Ouro

Em relacionamentos 1:N

A chave estrangeira (FK) fica SEMPRE do lado N (Muitos).

Exemplo: 1 Departamento tem N Funcionários. A tabela de Funcionário recebe a coluna id_departamento (FK).

Tabela DEPARTAMENTO
PK id_depto

Tabela FUNCIONARIO
PK cpf_func
FK id_depto (Aponta para Departamento)
Regra N:N

Em relacionamentos N:N

Ocorre a criação de uma Nova Tabela intermediária.

Exemplo: N Alunos em N Disciplinas. Surge a tabela "Matrícula" para conectar ambos.

Tabela ALUNO (PK id_aluno)
Tabela DISCIPLINA (PK cod_disc)

Tabela MATRICULA (Nova!)
FK id_aluno
FK cod_disc

3. Atributos Especiais

Nem todos os atributos são simples textos ou números. Alguns precisam de um tratamento especial ao transitar para o modelo lógico.

Atributo Composto

Um atributo que é formado por partes menores (ex: Endereço). No modelo lógico, ele desdobra-se em várias colunas na mesma tabela.

TB_CLIENTE (cpf, rua, cidade, codigo_postal)

Atributo Multivalorado

Um atributo que pode ter múltiplos valores (ex: Telefones). Fere as regras relacionais manter listas numa coluna. Ele vira uma nova tabela ligada por FK.

TB_TELEFONE (id_tel, numero, fk_cpf_cliente)

4. Comandos SQL Essenciais (DDL)

Para materializar o Modelo Lógico num SGBD real como o PostgreSQL, utilizamos a Linguagem de Definição de Dados (DDL). Abaixo estão os comandos basilares para estruturar o seu banco de dados.

CREATE TABLE

Cria uma nova tabela, definindo as suas colunas, tipos de dados e chaves de identificação.

CREATE TABLE cliente (
  id SERIAL PRIMARY KEY,
  nome VARCHAR(100)
);

ALTER TABLE

Modifica a estrutura de uma tabela já existente (adiciona colunas, altera tipos, define novas restrições).

ALTER TABLE cliente
ADD COLUMN email VARCHAR(50);

DROP TABLE

Elimina completamente uma tabela e todos os registos lá armazenados. Usar com extrema cautela!

DROP TABLE cliente;
