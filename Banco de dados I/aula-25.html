Aula 25: O Desafio TechNova

Crie, modifique, elimine e extraia dados no ciclo de vida completo de uma empresa real utilizando estritamente os comandos fundamentais de SQL.

BRIEFING

Bem-vindo à TechNova

A TechNova é uma startup em hipercrescimento. Você acaba de ser contratado como o novo DBA (Database Administrator). O seu antecessor deixou a infraestrutura inicial montada, mas a base de dados está completamente vazia.

O seu desafio é executar as 5 Missões do RH diretamente no seu painel do Neon.tech. Utilize os validadores abaixo para garantir que os seus comandos estão perfeitos antes de os executar na nuvem.

O Seu Ponto de Partida (DDL)

Copie e execute o script abaixo no seu Neon para preparar o terreno das missões.

Cuidado com o teclado: Certifique-se de que utiliza sempre aspas simples e retas ( ' ) para textos. Aspas curvas ou formatadas por processadores de texto gerarão erros de sintaxe no banco de dados.

00_setup_technova.sql
CREATE TABLE departamento ( id_depto SERIAL PRIMARY KEY, nome VARCHAR(50) NOT NULL UNIQUE ); CREATE TABLE funcionario ( id_func SERIAL PRIMARY KEY, nome VARCHAR(100) NOT NULL, cargo VARCHAR(60), salario NUMERIC(10,2), email VARCHAR(100) UNIQUE, ativo BOOLEAN DEFAULT TRUE, id_depto INT REFERENCES departamento(id_depto) ON DELETE RESTRICT ); -- Tabela para arquivar funcionários demitidos (Auditoria) CREATE TABLE funcionario_arquivo ( id_func INT, nome VARCHAR(100), cargo VARCHAR(60), salario NUMERIC(10,2), data_saida TIMESTAMP DEFAULT now() );

Missão 1: Fundar a Empresa (INSERT)

O sistema está no ar, mas a TechNova não tem departamentos nem empregados. Escreva dois comandos Bulk Insert (um para cada tabela).

  • Insira 5 departamentos à sua escolha (ex: 'TI', 'RH', 'Financeiro', 'Marketing', 'Projetos').
  • Insira 12 funcionários, distribuindo-os pelos departamentos (ids de 1 a 5). A funcionária 'Sol' (ID 12) ainda não tem e-mail configurado, portanto o valor dela deve ser explicitamente NULL.
  • Restrição do Arquiteto: É expressamente proibido omitir a declaração das colunas. Use sempre a forma segura: INSERT INTO tabela (col1, col2) VALUES (...).

Dica do Arquiteto: A Dominação do SERIAL

Se você errar um INSERT e tentar executá-lo novamente, a coluna SERIAL vai "pular" números (ex: do 1 salta para o 7). Para forçar a integridade e garantir que os seus IDs batem exatamente com o Snapshot abaixo, insira a coluna ID manualmente no seu comando (ex: INSERT INTO departamento (id_depto, nome) VALUES (1, 'TI')).

Missão 2: Correções e Promoções (UPDATE)

O RH enviou um e-mail urgente com 3 pedidos de atualização de dados. O Arquiteto Sênior avisou-o: "Antes de cada UPDATE, faça um Dry-Run usando SELECT para ter certeza de quem vai ser afetado!" Escreva as consultas SELECT primeiro, e logo a seguir os respectivos UPDATEs no mesmo terminal.

Atenção: O validador exigirá a presença dos comandos SELECT e a verificação exata dos IDs na sua resposta.

Pedido A: O funcionário com id_func = 1 teve o nome digitado errado. Atualize o nome para "Ana Sofia Silva".

Pedido B: O funcionário com id_func = 2 foi promovido! Altere o cargo para "Dev Sênior" e aumente o salário dele em 15% (dica: salario * 1.15).

Pedido C: Foi aprovado o reajuste anual da equipa de TI (id_depto = 1). Dê um aumento de 8% no salário de todos deste departamento de uma só vez.

m2_updates.sql
-- Pedido A
UPDATE funcionario SET nome = 'Ana Sofia Silva' WHERE id_func = 1;

-- Pedido B (Múltiplas colunas + Matemática relativa)
UPDATE funcionario 
SET cargo = 'Dev Sênior', salario = salario * 1.15 
WHERE id_func = 2;

-- Pedido C (Update em massa com FK)
UPDATE funcionario 
SET salario = salario * 1.08 
WHERE id_depto = 1;
3

Missão 3: Desligamentos e Auditoria

Infelizmente, dois funcionários deixaram a TechNova. Vamos aplicar as políticas de exclusão aprendidas na Aula 23.

A. O Soft Delete

O funcionário de id_func = 8 pediu demissão, mas ele tem e-mails atrelados ao histórico da empresa. Não podemos apagá-lo fisicamente. Desative-o no sistema (Soft Delete).

B. A Exclusão Atômica (CTE)

O funcionário de id_func = 11 estava no período de experiência e foi desligado. Use a magia das CTE (WITH ... AS) combinada com a cláusula RETURNING para o apagar definitivamente da tabela funcionario e inseri-lo instantaneamente na tabela funcionario_arquivo numa única query.

A Armadilha do Asterisco: A tabela de arquivo tem menos colunas do que a tabela original! Por isso, não pode usar RETURNING *. Tem de projetar explicitamente apenas as colunas correspondentes (id_func, nome, cargo, salario), caso contrário o PostgreSQL dará um erro fatal de "column mismatch".
m3_auditoria.sql
-- Parte A: Soft Delete (Usando UPDATE)
UPDATE funcionario SET ativo = FALSE WHERE id_func = 8;

-- Parte B: Exclusão com Arquivamento (CTE)
WITH funcionario_apagado AS (
    DELETE FROM funcionario 
    WHERE id_func = 11 
    RETURNING id_func, nome, cargo, salario
)
INSERT INTO funcionario_arquivo (id_func, nome, cargo, salario)
SELECT * FROM funcionario_apagado;
4

Missão 4: A Reestruturação (Integridade)

A diretoria decidiu que o departamento de "Projetos" (id_depto = 5) será dissolvido.

O Problema do RESTRICT

Se você tentar simplesmente apagar o departamento (DELETE FROM departamento WHERE id_depto = 5;) no Neon, o banco vai gritar um erro! A nossa FK foi configurada com ON DELETE RESTRICT. Temos um funcionário ainda lá (o 'Sol', id_func = 12).

A Solução do Arquiteto

Passo 1: Transfira todos os funcionários do departamento 5 para o departamento de "TI" (id_depto = 1) usando o comando UPDATE.
Passo 2: Agora que o departamento 5 está vazio, proceda com o comando DELETE para obliterar o departamento.

m4_demolicao.sql
-- Passo 1: Esvaziar o departamento movendo os dependentes
UPDATE funcionario SET id_depto = 1 WHERE id_depto = 5;

-- Passo 2: O escudo RESTRICT não o impedirá agora
DELETE FROM departamento WHERE id_depto = 5;

Missão 5: Relatório da Diretoria (SELECT)

A poeira baixou. Agora a diretoria exige relatórios da situação atual da empresa. Utilize a Projeção DQL (Aula 24) para criar as consultas abaixo.

Relatório 1: Liste apenas os nomes e cargos de todos os funcionários.

Relatório 2: Crie uma lista com os cargos da empresa sem repetições (Dica: Eliminação de Ruído).

Relatório 3: Mostre o nome e uma coluna calculada multiplicando o salário por 12. Apelide essa nova coluna de salario_anual (Dica: Coluna Virtual).

Relatório 4: Exiba o nome todo em maiúsculas (UPPER) e o email usando o escudo contra nulos (COALESCE) substituindo vazios por 'Sem Email'.

Nota: Antes de validar o código aqui, execute-o no Neon e maravilhe-se a ver o COALESCE substituir o NULL da funcionária Sol por 'Sem Email'!

Dica do Neon: Se executar vários SELECTs ao mesmo tempo, o Neon só mostrará a tabela do último. Para ver os relatórios separadamente, selecione (sublinhe com o rato) um comando de cada vez antes de clicar em Run.

m5_relatorios.sql
-- R1
SELECT nome, cargo FROM funcionario;

-- R2
SELECT DISTINCT cargo FROM funcionario;

-- R3
SELECT nome, salario * 12 AS salario_anual FROM funcionario;

-- R4
SELECT UPPER(nome), COALESCE(email, 'Sem Email') FROM funcionario;
