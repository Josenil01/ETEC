2. O Papel e a Segurança

DBA (Administrator)

Garante o Princípio do Menor Privilégio: cada usuário tem apenas o acesso estritamente necessário à sua função. Se o leigo só precisa ler, o DBA bloqueia o comando DELETE para ele.

Ferramenta: CLI / Terminal / Cloud Console (Neon)

Analistas de Sistemas

Mapeiam o Mini-mundo. Traduzem regras de negócio em diagramas técnicos que o DBA irá implementar. Focam na arquitetura da informação.

Ferramenta: CASE Tools / SQL Editor / Draw.io

Programadores

Consomem o banco através de código. Fazem o INSERT de novos registros e os UPDATES através da API da aplicação.

Ferramenta: VS Code / IDEs / Postman

Usuários Leigos

O utilizador comum. Para ele, o banco é invisível. Ele interage apenas com a interface gráfica (UI) desenhada pelo programador.

Ferramenta: Navegador Web / Mobile App UI

Segurança Ativa

Incidentes reportados no Slack da TechNova. Identifique o profissional responsável.

"O servidor está lento e precisamos restaurar o backup de ontem!"

"Preciso que a tela de cadastro de produtos aceite fotos em HD."

"Precisamos modelar uma nova tabela para gerir fornecedores."

"Alguém apagou os clientes porque tinha acesso a mais! Quem reconfigura as permissões?"

PROTOCOLO DE SEGURANÇA VALIDADO! Você domina as responsabilidades do ecossistema.
