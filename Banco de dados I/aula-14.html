O "Início a Frio" (Cold Start)

Como o Neon é gratuito e econômico, ele "pausa" seu servidor se ninguém o usa por 5 minutos. A primeira consulta do dia pode demorar 3-5 segundos para "acordar" o motor. Tenha paciência!

Connection Pooling

O Neon oferece duas URLs. Uma termina em ?sslmode=require e a outra tem um modo de "Pooler". Use o Pooler para apps Web (Vercel/Next.js) para evitar conexões travadas.

2. Anatomia da Dashboard

Ao entrar no neon.tech, estas são as bússolas que guiarão você:

Compute Endpoint

O endereço lógico do seu hardware virtual. Ele define quanta CPU e RAM seu banco está usando no momento.

SQL Editor

Sua aba principal. Onde criaremos tabelas e faremos consultas sem precisar de softwares externos instalados.

Branching

Crie uma "cópia paralela" do banco para testar alterações arriscadas sem estragar os dados de produção.

Region

A localização física (ex: AWS Ohio). Escolha sempre a mais próxima do seu servidor de aplicação para evitar lag.

Protocolo de Segurança

Sua Connection String é o "CPF" do seu banco. Se alguém a tiver, pode apagar todos os seus dados.

# O QUE NUNCA FAZER:
const db = "postgresql://user:senha123@ep-shiny-cloud.us-east-2.neon.tech/neondb";

# O QUE ARQUITETOS FAZEM (Arquivo .env):
DATABASE_URL=postgresql://user:senha123@ep-shiny-cloud...
// O código lê o arquivo, mas o arquivo nunca vai para o GitHub!

Quiz de Arquiteto

Rápido

Um cliente reclama que o sistema "travou" ao abrir pela primeira vez de manhã, mas depois funcionou rápido. Qual o motivo técnico?

Resposta: É o Cold Start (Início a Frio). O servidor do Neon estava "dormindo" para economizar recursos e demorou alguns segundos para carregar o motor do banco de dados na memória.
