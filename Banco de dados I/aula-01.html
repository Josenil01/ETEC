O Superpoder: Metadados

O Dado sobre o Dado

Imagine encontrar um papel com o número "25". Sem a etiqueta, ele é lixo. O Metadado é o mapa que diz que aquela gaveta só aceita números e que o nome dela é "Temperatura_Celsius". No SQL, a DDL (CREATE TABLE) é onde definimos os metadados, e a DML (INSERT) é onde inserimos os dados.

O Elo do Arquiteto

"O Arquiteto define o Metadado (A forma) para que a aplicação gere Informação (A mensagem)."

3. A Lente do Mini-Mundo

Universo de Discurso

O Arquiteto como Escultor

O mundo real é infinito e cheio de detalhes irrelevantes. O Mini-Mundo é a parte da realidade que removemos do mármore para criar o software.

Combater a Ambiguidade

A linguagem humana é vaga. Quando um cliente diz "venda", ele pode referir-se a uma "transação", um "pedido" ou uma "entrega". As Entrevistas e o levantamento servem para remover esta ambiguidade. No banco de dados, não há espaço para "talvez".
