Módulo 6: Linguagem de Consulta (DQL) Arquiteto de Banco de Dados

Aula 24: O Oráculo

Introdução à DQL e Projeção: A arte de questionar o banco de dados e extrair apenas a informação que realmente importa.

1. O Despertar do Oráculo

Já construímos a estrutura (DDL) e já manipulamos as informações lá dentro (DML). Porém, 90% do tempo de um sistema em produção (como o ecrã do seu banco ou o feed do Instagram) é gasto a ler dados.

Apresento-lhe a DQL (Data Query Language). É com ela que fazemos perguntas à base de dados. Ao contrário das outras linguagens, a DQL não altera, não destrói e não cria nada. Ela apenas observa e devolve os resultados. E o seu rei indiscutível é o comando SELECT.

Anatomia Base
SELECT coluna1, coluna2

1. O QUE EU QUERO LER? (A Projeção)

FROM tabela_alvo;

2. A Magia da Projeção

O Pecado do Asterisco (*)

Muitos iniciantes usam sempre SELECT * FROM tabela;. O asterisco significa "Traga-me TODAS as colunas".

Porque é mau em produção? Se a tabela tiver 50 colunas (incluindo fotos gigantes em base64 ou senhas encriptadas) e você só precisar do nome do utilizador para mostrar no cabeçalho do site, estará a desperdiçar memória, rede e dinheiro do servidor atoa.
SELECT * FROM funcionario;

A Projeção Cirúrgica

Projeção é o ato técnico de escolher exatamente quais colunas queremos visualizar. O banco constrói uma "tabela virtual" instantânea apenas com os dados que pedimos, descartando o resto.

Para projetar, basta trocar o asterisco pelo nome das colunas separadas por vírgula. É mais seguro, mais rápido e mais profissional.
SELECT nome, cargo FROM func;

3. Superpoderes da Projeção

O Silenciador (DISTINCT)

Se projetar apenas os cargos da empresa, o banco vai repetir "Dev", "Dev", "Dev" por cada funcionário que tiver esse cargo. A cláusula DISTINCT elimina o ruído e devolve apenas os valores únicos.

SELECT DISTINCT cargo FROM func;

Colunas Virtuais (AS)

O SELECT também é uma calculadora! Podemos aplicar matemática (ex: multiplicar salário por 12) e ainda "apelidar" essas novas colunas com a palavra AS para o relatório ficar legível.

SELECT salario * 12 AS ganho_anual FROM func;

4. A Forja de Dados na Projeção

Os Arquitetos não se limitam a ler dados; eles transformam-nos em tempo real antes de os enviar para a aplicação. Conheça as três ferramentas avançadas de formatação:

A Lógica (CASE WHEN)

É o "If/Else" do SQL. Permite criar colunas com base em condições complexas, poupando processamento no Back-end.

SELECT nome,
CASE
  WHEN salario > 5000 THEN 'Sénior'
  ELSE 'Júnior'
END AS nivel
FROM func;

Escudo Nulo (COALESCE)

Valores Nulos (NULL) podem crashar a aplicação. O COALESCE deteta o vazio e projeta um valor padrão no seu lugar.

SELECT nome,
COALESCE(cargo, 'Sem Cargo')
AS cargo_seguro
FROM func;

A Costura ( || )

Une múltiplas colunas e textos estáticos numa única super-coluna. Perfeito para gerar relatórios prontos.

SELECT
nome || ' atua como ' || cargo
AS descricao
FROM func;

5. Laboratório DQL: A Consola do Oráculo

Dica: Ctrl+Alt+X

A base de dados das Aulas 21 a 23 foi restaurada (10 funcionários). O seu terminal agora age como um verdadeiro motor DQL. Escreva as consultas e o sistema irá projetar os resultados em tempo real à direita.

1

O Reconhecimento Base

Para começar, precisamos de ver como está a estrutura completa da tabela de departamento. Utilize o famigerado * (asterisco) para projetar todas as colunas disponíveis.

SELECT * FROM departamento;
2

A Arte da Projeção

O setor de Recursos Humanos pediu a lista de todos os empregados. Mas eles não querem ver salários nem IDs. Eles pediram estritamente o nome e o cargo de toda a gente da tabela funcionario.

SELECT nome, cargo FROM funcionario;
3

A Eliminação do Ruído

A direção quer saber que cargos existem atualmente na empresa, sem repetições. Utilize SELECT DISTINCT cargo FROM funcionario; para extrair apenas a matriz única.

SELECT DISTINCT cargo FROM funcionario;
4

A Coluna Virtual

O departamento financeiro precisa de planear o próximo ano. Extraia o nome do funcionário e crie uma coluna virtual multiplicando o salario por 12. Apelide essa nova coluna de salario_anual usando a cláusula AS.

SELECT nome, salario * 12 AS salario_anual FROM funcionario;
5
Ultimate Boss

O Relatório do Diretor

O Diretor quer uma lista formatada. Projete o nome do funcionário. Use o COALESCE no cargo para preencher com 'Sem Cargo' caso seja nulo. Por fim, use o CASE WHEN no salario para classificar salários > 4000 como 'Alto', e os restantes como 'Normal', apelidando essa coluna de classificacao.

SELECT nome, COALESCE(cargo, 'Sem Cargo') AS cargo_seguro, CASE WHEN salario > 4000 THEN 'Alto' ELSE 'Normal' END AS classificacao FROM funcionario;
