Módulo 5: Manipulação de Dados (DML) Arquiteto de Base de Dados

Aula 22: O Cirurgião

DML e Atualização de Dados: Como corrigir, promover e alterar registos existentes com precisão cirúrgica utilizando o UPDATE.

1. A Arte da Precisão

Na Aula 21, aprendemos a dar vida à base de dados criando registos novos com o INSERT. Contudo, os dados não são estáticos. Um funcionário muda de morada, é promovido ou reparamos num erro de digitação no seu nome.

Não demolimos uma casa inteira só para pintar uma parede. Da mesma forma, não apagamos (DELETE) um registo e o recriamos (INSERT) apenas para corrigir uma letra. Utilizamos o Bisturi do Arquiteto: o comando UPDATE.

2. A Sintaxe da Mutação

O comando UPDATE requer três coordenadas cruciais: a tabela alvo, as colunas que vão sofrer mutação e a condição que filtra quem será afetado.

Estrutura do Comando
UPDATE funcionario

1. Qual é a tabela que vamos operar?

SET cargo = 'Senior', salario = 6000

2. O que vamos alterar? (Use vírgulas para alterar múltiplas colunas)

WHERE id_func = 2;

3. A Condição (Filtro): QUEM vai sofrer esta alteração?

Atualizações Relativas

Muitas vezes não sabemos o valor final, mas sim a operação matemática. Podemos usar o valor atual da coluna no SET para dar um aumento de 10%, por exemplo.

aumento.sql
UPDATE funcionario
SET salario = salario * 1.10
WHERE cargo = 'Junior';

O Retorno Imediato (RETURNING)

Tal como no INSERT, o Back-end (Node.js/Python) precisa muitas vezes de saber qual foi o novo valor calculado para o exibir no ecrã sem ter de fazer um SELECT adicional.

api_response.sql
UPDATE funcionario
SET cargo = 'Pleno'
WHERE id_func = 3
RETURNING salario, cargo;

3. O Pesadelo do Arquiteto

A cláusula WHERE é opcional na sintaxe do SQL. O motor da base de dados não o impedirá de enviar um comando sem ela. Mas o que acontece se executar a mutação abaixo?

desastre.sql
-- Promovendo o João a CEO? Não. Promovendo a empresa inteira!
UPDATE funcionario
SET cargo = 'CEO', salario = 500000;

Efeito Colateral Crítico:

Sem o WHERE, o bisturi transforma-se numa bomba. O banco de dados vai ler a tabela desde a primeira linha até à milionésima, alterando o cargo e o salário de TODOS OS FUNCIONÁRIOS para CEO. Milhares de carreiras e folhas de pagamento destruídas num milissegundo. Nunca se esqueça do WHERE.

4. O Teste a Seco (Dry Run)

O Truque do DBA Sénior

Para nunca cometer o erro de atualizar as linhas erradas, os administradores de bases de dados (DBAs) usam sempre um Teste a Seco. A regra de ouro é simples: um arquiteto nunca escreve o UPDATE diretamente.

Passo 1

Escreva um SELECT com a condição exata que pretende usar. Analise se as linhas devolvidas são APENAS as que deseja afetar.

SELECT * FROM funcionario
WHERE id_depto = 5;
Passo 2

Se os dados estiverem corretos, mantenha o WHERE intacto e substitua apenas a primeira linha pelo UPDATE e SET.

UPDATE funcionario
SET bonus = 100
WHERE id_depto = 5;

5. O Elo com a Chave Primária

4. O Elo com a Chave Primária

Mesmo lembrando de usar o WHERE, pode cometer erros de precisão. Imagine que quer atualizar o salário da "Ana".

... WHERE nome = 'Ana';

O Perigo: Se a sua empresa tiver 5 pessoas chamadas "Ana", todas elas receberão a alteração. Para cirurgias que afetam uma única pessoa, o Arquiteto usa SEMPRE a Chave Primária (PK) no filtro. A PK é matemática pura: é impossível afetar mais do que um registo.

... WHERE id_func = 1; -- 100% Seguro. Só existe um ID 1.

6. Laboratório Cirúrgico

Dica: Ctrl+Alt+X

Restaurámos a base de dados da Aula 21 no visualizador à direita. Os 10 funcionários e os departamentos estão no sistema. Assuma a consola e execute as atualizações solicitadas nos tickets do RH.

1

Correção de Digitação

O funcionário com o ID 1 (Ana) teve o seu nome inserido incompleto. Atualize o nome dela para 'Ana Silva' utilizando a sua Chave Primária.

UPDATE funcionario SET nome = 'Ana Silva' WHERE id_func = 1;
2

A Promoção do Rui

O funcionário Rui (ID 2) foi promovido. Atualize o cargo dele para 'Tech Lead' e aumente o salario para 6500 na mesma instrução (separando por vírgula).

UPDATE funcionario SET cargo = 'Tech Lead', salario = 6500 WHERE id_func = 2;
3
Boss

Atualização em Massa (Segura)

A direção aprovou um bónus! O departamento de Vendas (id_depto = 5) bateu a meta. Atualize o salario de TODOS os funcionários deste departamento para 5000 de uma só vez usando o filtro de FK.

UPDATE funcionario SET salario = 5000 WHERE id_depto = 5;
4
Ultimate Boss

Aumento no Marketing (Relativo + RETURNING)

Dê um aumento de 10% a todos os funcionários do departamento de Marketing (id_depto = 4) multiplicando o salário atual por 1.10. Por fim, adicione a cláusula mágica para devolver os novos salários (RETURNING salario) ao Back-end.

UPDATE funcionario SET salario = salario * 1.10 WHERE id_depto = 4 RETURNING salario;
