Aula 29: A Grande Ponte

Inner Join: Aprenda a unir tabelas que foram separadas na normalização e transforme IDs técnicos em informações legíveis.

1. O Porquê do Join

Nas aulas de **Normalização**, aprendemos que repetir o nome do departamento em cada linha do funcionário era um erro. Por isso, separámos os dados em várias tabelas.

Mas para um humano, ler "ID 1" não significa nada. Queremos ler "TI". O INNER JOIN reconecta as ilhas de dados seguindo as Chaves Estrangeiras para buscar os nomes reais.

FUNCIONARIO
fk_depto = 1
ON
DEPARTAMENTO
id_depto = 1

O banco cruza as linhas onde os valores das chaves coincidem.

2. A Sintaxe do Cruzamento

Para fazer um Join, declaramos a Tabela A, usamos o comando INNER JOIN para a Tabela B e a cláusula ON para definir a ponte.

SELECT f.nome AS colaborador, d.nome AS setor
FROM funcionario f
INNER JOIN departamento d ON f.id_depto = d.id_depto;

O Problema da Ambiguidade

Se ambas as tabelas tiverem uma coluna nome, o banco dará erro. Use Aliases (f, d) para distinguir a origem.

Múltiplas Condições

Nota de Arquiteto: A cláusula ON aceita múltiplas condições usando AND (ex: ON f.id_depto = d.id_depto AND f.status = 'Ativo'), embora o padrão seja apenas o par de chaves.

3. Central de Chamados QuantumShop

Dica: Ctrl+Alt+X

Interprete as necessidades dos departamentos e reconecte as matrizes de dados. As colunas terão cores diferentes conforme a sua origem.

1

Ticket #201: RH (Nomenclatura)

"Arquiteto, queremos o nome de todos os funcionários ao lado dos seus setores. Use o alias colaborador para a pessoa e setor para o departamento."

SELECT f.nome AS colaborador, d.nome AS setor FROM funcionario f INNER JOIN departamento d ON f.id_depto = d.id_depto;
2

Ticket #202: Marketing

"Traga-me apenas o nome e o salário de quem trabalha no setor de 'Marketing'."

SELECT f.nome, f.salario FROM funcionario f INNER JOIN departamento d ON f.id_depto = d.id_depto WHERE d.nome = 'Marketing';
3

Ticket #203: CFO (Resumo BI)

"Gere a Média Salarial (AVG) agrupada pelo Nome do Departamento. Apelide a média de 'custo_medio'."

SELECT d.nome, AVG(f.salario) AS custo_medio FROM funcionario f INNER JOIN departamento d ON f.id_depto = d.id_depto GROUP BY d.nome;
4

Ticket #204: CTO (Mapa)

"Liste o Nome do Funcionário, o Nome do Departamento e a Cidade onde ele atua. Cruza as três tabelas (f, d, l)!"

SELECT f.nome, d.nome, l.cidade FROM funcionario f INNER JOIN departamento d ON f.id_depto = d.id_depto INNER JOIN local l ON d.fk_local = l.id;
5

Ticket #205: A Ponte Trocada

"Um estagiário tentou unir o Nome e o Departamento, mas ligou o ID do funcionário ao ID do depto (f.id_func = d.id_depto)! Conserte o ON para as colunas corretas."

SELECT f.nome, d.nome FROM funcionario f INNER JOIN departamento d ON f.id_depto = d.id_depto;
