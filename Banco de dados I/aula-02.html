O Caos dos Arquivos

Entenda as falhas estruturais, o custo da resposta e a vulnerabilidade que forçaram a criação dos SGBDs.

1. A Era das Ilhas de Dados

Antigamente, as empresas migravam do papel para o computador usando Sistemas de Arquivos isolados. Sem uma inteligência central, cada software falava a sua própria língua, criando silos de informação inacessíveis entre si.

CADASTRO_RH.txt
// Dados do Pessoal
ID: 101 | NOME: Rui Costa
MORADA: Rua de Lisboa, 5
TEL: 912345678
VENDAS.csv
// Registos Comerciais
ID: 101 | NOME: Rui Costa
SALARIO: 2500€
TEL: 912345678

2. As Patologias dos Arquivos

Redundância

Repetição desnecessária. Ocupa espaço e gera esforço duplo para qualquer alteração simples.

Inconsistência

Dados divergentes. Qual é a verdade quando o RH atualiza um dado mas as Vendas mantêm o antigo?

Acesso Concorrente

Dois utilizadores a gravar ao mesmo tempo destroem o ficheiro. O último a gravar "atropela" o primeiro.

Dependência Programa-Dados

O software é escravo do formato físico. Se mudares a ordem das colunas, a aplicação crasha.

Vulnerabilidade

O controlo de acesso é rudimentar. Qualquer pessoa com acesso à pasta pode apagar tudo ou ver salários.

Inflexibilidade de Acesso

Qualquer nova pergunta do negócio exige um novo programa escrito do zero para ler o ficheiro e filtrar os dados.

3. Laboratório: Teste de Stress Analógico

Controlo de Crise

Como Arquiteto, deves experienciar o perigo. Testa as falhas estruturais abaixo:

RH_DB.csv
ID: 101
Nome: Rui Costa
Telefone:
VENDAS_DB.csv Cliente: Rui Costa
Total Gasto: 1.200€
Telefone: 912345678
ALERTA: DADOS INCONSISTENTES DETECTADOS!
FATAL ERROR: Escrita interrompida. Ficheiro RH_DB.csv corrompido. Falta de atomicidade.
Diagnóstico do Arquiteto: Mudar o telefone criou uma Inconsistência. A falha de energia provou a falta de Atomicidade. Além disso, note que cruzar os dados destes dois ficheiros num sistema antigo exigiria um terceiro programa complexo para unir os dados em memória RAM, algo que o SGBD fará automaticamente com um simples JOIN.

4. O Herói: O SGBD

O SGBD (Sistema de Gerenciamento de Banco de Dados) não organiza apenas os dados; ele fornece Independência e Velocidade.

O Custo da Resposta

Lado A: Arquivos (Programação Manual)
file = open('rh.txt');
while (line = file.read()) {
  parts = line.split(',');
  if (parts[3] > 2000) {
    print(parts[1]);
  }
}
// +30 linhas de tratamento de erro
Lado B: SGBD (SQL Declarativo)
SELECT nome FROM funcionario
WHERE salario > 2000;

O SGBD não guarda apenas os dados; ele entende a estrutura e responde por si próprio.

Antigo: Dependência

Software e Estrutura Física estão amarrados. Mudar um bit no disco quebra a aplicação inteira.

Novo: Independência

O SGBD abstrai o disco. Podes mudar a "arrumação" interna dos dados sem reescrever uma única linha do software.
