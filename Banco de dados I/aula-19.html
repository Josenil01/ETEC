Etapa 3: Gênesis DDL (Aulas 15, 16)

Chegou a hora de escrever o código. Crie as 4 tabelas definidas na Etapa 2.
Atenção à restrição: NÃO utilize as chaves estrangeiras (REFERENCES) agora. Apenas defina as chaves primárias (SERIAL PRIMARY KEY) e garanta que o email seja UNIQUE e obrigatório.

genesis.sql
solucao_etapa3.sql
CREATE TABLE cliente (
    id_cliente SERIAL PRIMARY KEY,
    nome VARCHAR(100) NOT NULL,
    email VARCHAR(150) NOT NULL UNIQUE
);

CREATE TABLE produto (
    id_produto SERIAL PRIMARY KEY,
    nome VARCHAR(150) NOT NULL,
    preco_atual DECIMAL(10,2)
);

CREATE TABLE pedido (
    id_pedido SERIAL PRIMARY KEY,
    id_cliente INT,
    data_pedido DATE DEFAULT CURRENT_DATE
);

CREATE TABLE item_pedido (
    id_pedido INT,
    id_produto INT,
    quantidade INT,
    preco_historico DECIMAL(10,2),
    PRIMARY KEY (id_pedido, id_produto)
);

Etapa 4: Conexão e Evolução (Aula 17)

Com as ilhas criadas, o CEO fez exigências finais:
1. Conecte o pedido ao cliente garantindo que não possamos apagar clientes que tenham pedidos (RESTRICT).
2. Adicione uma coluna chamada status à tabela de pedido.
3. Conecte a tabela item_pedido à tabela pedido com física de cascata (CASCADE).

solucao_etapa4.sql
-- 1. Ponte Segura (Restrict)
ALTER TABLE pedido 
ADD CONSTRAINT fk_pedido_cliente 
FOREIGN KEY (id_cliente) REFERENCES cliente(id_cliente) ON DELETE RESTRICT;

-- 2. Nova Coluna
ALTER TABLE pedido 
ADD COLUMN status VARCHAR(50);

-- 3. Reação em Cadeia (Cascade)
ALTER TABLE item_pedido 
ADD CONSTRAINT fk_item_pedido_ped 
FOREIGN KEY (id_pedido) REFERENCES pedido(id_pedido) ON DELETE CASCADE;
