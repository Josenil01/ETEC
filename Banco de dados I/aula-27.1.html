Funções Agregadas: A arte de consolidar milhões de linhas em números estratégicos para Dashboards e Relatórios.

1. De Detalhes para Dashboards

Até agora, aprendemos a ler e filtrar funcionários um a um. Mas o CEO de uma empresa não quer ler uma lista de 5.000 nomes; ele quer saber: "Quanto gastamos em salários por mês?" ou "Qual é a média de vendas do dia?".

É aqui que entram as Funções Agregadas. Elas processam um conjunto de dados (múltiplas linhas) e devolvem um resultado único. Elas são a base de qualquer sistema de Business Intelligence (BI).

2. O Arsenal de Cálculo

COUNT(*)

O contador. Devolve a quantidade total de linhas que atendem ao critério.

SELECT COUNT(*) FROM funcionario;

SUM(coluna)

O somatório. Soma todos os valores numéricos de uma coluna específica.

SELECT SUM(salario) FROM func;

AVG(coluna)

A média (Average). Soma os valores e divide pela quantidade de linhas.

SELECT AVG(salario) FROM func;

MIN / MAX

Os extremos. Encontram o menor e o maior valor de uma coluna.

SELECT MIN(salario), MAX(salario) FROM func;

A Armadilha do Vazio: COUNT(*) vs. COUNT(coluna)

Existe uma diferença vital! O COUNT(*) conta todas as linhas da tabela, mesmo que contenham nulos. Já o COUNT(coluna) (e funções como SUM e AVG) ignoram os valores NULL. Se um funcionário não tem cargo definido, ele será contado pelo asterisco, mas ignorado pelo contador de cargos!

Dica de Arquiteto: O uso de Alias (AS)

Quando usa uma função agregada, o banco não sabe que nome dar à coluna (muitas vezes chama-lhe apenas "count" ou "avg"). Utilize sempre o AS para dar um nome legível ao seu relatório, ex: SELECT AVG(salario) AS media_salarial FROM funcionario;

O "Mix" Proibido

Um erro clássico é tentar fazer SELECT nome, SUM(salario) FROM funcionario;. O banco dará ERRO. Porquê? Porque o "nome" é um detalhe individual (várias linhas) e o "SUM" é um resumo (uma linha). Não podemos misturar o micro com o macro no mesmo comando sem o Poder do Agrupamento (GROUP BY), que estudaremos na próxima aula!
