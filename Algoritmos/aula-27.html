1. Pilares da Proteção

Private

Tranca o atributo. Apenas a própria classe pode ler ou alterar o valor desta "caixa".

Public

Abre o atributo. Qualquer parte do reino (projeto) pode ver e mudar o valor livremente.

Getters

Métodos de leitura. Funcionam como "vigias" que mostram o valor guardado sem deixar ninguém tocar.

Setters

Métodos de escrita. Permitem mudar o valor, mas com regras e validações de segurança.

2. Laboratório: O Cofre de Segurança Máxima

"Não basta guardar o ouro; é preciso garantir que ele não ultrapasse a capacidade do baú e que o dono seja devidamente identificado."

01 CofreReal.java (A Classe com Regras de Negócio)

CofreReal.java 
class CofreReal {
    // ATRIBUTOS TRANCADOS (Private)
    private String dono;
    private int moedas;
    private final int CAPACIDADE_MAXIMA = 5000;

    // GETTERS: Acesso de leitura
    public String getDono() { return dono; }
    public int getMoedas() { return moedas; }

    // SETTERS: Escrita com proteção extrema
    public void setDono(String novoDono) {
        // Validação: Nome não pode ser vazio ou muito curto
        if (novoDono != null && novoDono.length() >= 3) {
            this.dono = novoDono;
        } else {
            System.out.println("⚠️ Erro: Nome do dono inválido!");
        }
    }

    public void adicionarOuro(int quantidade) {
        if (quantidade <= 0) {
            System.out.println("⚠️ Erro: Quantidade deve ser positiva!");
        } else if (moedas + quantidade > CAPACIDADE_MAXIMA) {
            System.out.println("🚨 Alerta: O cofre vai transbordar! Limite é " + CAPACIDADE_MAXIMA);
        } else {
            moedas += quantidade;
            System.out.println("💰 " + quantidade + " moedas guardadas com sucesso.");
        }
    }

    public void statusDoCofre() {
        System.out.println("\n=== RELATÓRIO DO COFRE ===");
        System.out.println("PROPRIETÁRIO: " + (dono == null ? "Desconhecido" : dono));
        System.out.println("SALDO ATUAL: " + moedas + " moedas");
        System.out.println("ESPAÇO LIVRE: " + (CAPACIDADE_MAXIMA - moedas));
    }
}

02 BancoCentral.java (O Executor)

BancoCentral.java 
public class BancoCentral {
    public static void main(String[] args) {
        CofreReal cofreArthur = new CofreReal();

        // 1. Tentativa de atribuir nome inválido
        cofreArthur.setDono("Ar"); 
        
        // 2. Atribuição correta
        cofreArthur.setDono("Arthur Pendragon");

        // 3. Tentativas de depósito
        cofreArthur.adicionarOuro(2000); // OK
        cofreArthur.adicionarOuro(-100);  // Erro: Valor negativo
        cofreArthur.adicionarOuro(4000); // Alerta: Transborda a capacidade

        // 4. Tentativa de hackear o acesso (Vai dar erro de compilação)
        // cofreArthur.moedas = 10000; // O compilador impede o acesso direto!

        // 5. Exibição final
        cofreArthur.statusDoCofre();
    }
}

3. Erros Arcanos: "has private access"

Acesso Direto Inválido

Tentar ler ou escrever heroi.vida = 10; quando o atributo é private causa erro de compilação. Lembra-te: tens de usar os métodos get/set.

Esquecer de retornar no Get

O método get precisa de ter o comando return. Se o deixares vazio, quem chamou o método não receberá a informação.
