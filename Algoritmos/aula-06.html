Aula 06: As Encruzilhadas do Destino

Dê "cérebro" ao seu herói. Aprenda a usar as estruturas de decisão para que o seu código possa escolher caminhos baseados em condições.

1. O Poder da Escolha

Até agora, os nossos códigos eram lineares: executavam uma linha após a outra sem questionar. As Estruturas de Decisão (ou Condicionais) permitem que o programa avalie uma pergunta (condição) e execute blocos de código diferentes dependendo da resposta ser Verdadeira ou Falsa.

⚡ Consulta Rápida: Operadores Relacionais

Antes de criar condições, o seu código precisa saber como comparar valores. O computador usa estes símbolos para criar perguntas que retornam Verdadeiro ou Falso.

== Igual a
!= Diferente de
> Maior que
< Menor que
>= Maior ou igual
<= Menor ou igual

🔗 Consulta Rápida: Operadores Lógicos

Às vezes, uma condição não basta. Os operadores lógicos permitem juntar múltiplas perguntas numa só encruzilhada.

e Todas têm de ser verdadeiras
ou Basta uma ser verdadeira
nao Inverte a resposta (falso vira verdadeiro)
Exemplo: se (nivel > 10 e tem_chave == verdadeiro)

2. A Estrutura Se...Senao

Estrutura Simples (Se) "Se eu tiver moedas, compro a espada."

Executa um bloco apenas se a condição for verdadeira. Se for falsa, ignora completamente e segue em frente.

Estrutura Composta (Se...Senao) "Se tiver mana, lanço fogo. Senão, ataco."

Garante uma bifurcação exata. Oferece um caminho alternativo obrigatório caso a primeira condição seja falsa.

3. Laboratório Prático de Decisões

01 O Portão da Mana

Cenário: O utilizador introduziu que tem 50 de mana inicial. Analise o código e preencha o rastro da variável mana_atual.

feitico_mana.por
inteiro mana_atual = 50
inteiro CUSTO = 40

se (mana_atual >= CUSTO) {
  escreva("Feitiço lançado! ✨")
  mana_atual = mana_atual - CUSTO
} senao {
  escreva("Mana insuficiente... 🌑")
}
Momento Condição (>= 40) mana_atual
Início - 50
Teste do 'se' 50
Fim do Bloco -

03 A Jornada de Ranks (Encadeamento)

Dica do Oráculo

Em um bloco se...senao se, o computador é muito eficiente (e um pouco preguiçoso). Ele testa de cima para baixo e executa apenas O PRIMEIRO caminho verdadeiro que encontrar. Assim que acha a verdade, ele ignora todo o resto do bloco encadeado e salta para o fim.

Cenário: Um herói submeteu uma ficha com XP = 4500. Ajude o sistema a rastrear qual bloco será ativado.

rank_heroi.por
inteiro xp = 4500

se (xp < 1000) {
  escreva("Rank: Bronze 🛡️")
} senao se (xp < 3000) {
  escreva("Rank: Prata ⚔️")
} senao se (xp < 6000) {
  escreva("Rank: Ouro 🏆")
} senao {
  escreva("Rank: Lendário 💎")
}
Condição Avaliada Resultado
xp < 1000
xp < 3000
xp < 6000
Qual Rank ele recebeu?

04 O Baú do Tesouro (Decisões Aninhadas)

Podemos ter um se dentro de outro se! É o que chamamos de condições aninhadas. Primeiro você passa pela porta da rua, para depois escolher o quarto. Cenário: Tem a chave, mas ela é prata.

bau_tesouro.por
logico tem_chave = verdadeiro
cadeia cor_chave = "prata"
cadeia premio = "nenhum"

se (tem_chave == verdadeiro) {
  se (cor_chave == "dourada") {
    premio = "1000 moedas"
  } senao {
    premio = "pocao de cura"
  }
} senao {
  escreva("O baú está trancado.")
}
Avaliação Passo a Passo Resultado
tem_chave == verdadeiro
cor_chave == "dourada"
Qual foi o premio final?

Nível: Aprendiz

01

Par ou Ímpar: Peça um número ao utilizador e diga se ele é par ou ímpar (Dica: use o resto da divisão por 2).

inteiro num
leia(num)

se (num % 2 == 0) {
  escreva("Par")

  escreva("Ímpar")
}

leia(nota)

se (nota >= 7) {
  escreva("Aprovado")

Nível: Cavaleiro

03

Duelo de Titãs: Leia dois números diferentes e exiba qual deles é o maior.

inteiro n1, n2
leia(n1, n2)

se (n1 > n2) {
  escreva(n1, " é o maior")
} senao se (n2 > n1) {
  escreva(n2, " é o maior")
} senao {
  escreva("São iguais")
}
04

Ano Bissexto: Crie um programa que leia um ano e verifique se ele é bissexto (divisível por 4).

inteiro ano
leia(ano)

se (ano % 4 == 0) {
  escreva("Bissexto")
} senao {
  escreva("Não é bissexto")
}

Nível: Mestre Arcano

05

Calculadora de Runas: Peça dois números e um caracter (+, -, *, /). Use estruturas de decisão para realizar a operação matemática correta baseada no símbolo digitado.

real n1, n2
caracter op
leia(n1, n2, op)

se (op == '+') {
  escreva(n1 + n2)
} senao se (op == '-') {
  escreva(n1 - n2)
} senao se (op == '*') {
  escreva(n1 * n2)
} senao se (op == '/') {
  escreva(n1 / n2)
}
