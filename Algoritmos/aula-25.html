📚 O Novo Paradigma: Porquê POO?

Até agora, programávamos de forma procedural: o código era uma lista de ordens e as variáveis (nome, hp, ouro) flutuavam sozinhas pelo programa. Em sistemas reais, isso vira um caos.

A Programação Orientada a Objetos (POO) resolve isso agrupando dados e comportamentos. Em vez de ter "variáveis de um herói", tu crias um Tipo Herói.

Antes (Variáveis Soltas) String nome1 = "Arthur";
int hp1 = 100;
String nome2 = "Elena";
int hp2 = 80;
Agora (Objetos) Heroi h1 = new Heroi();
h1.nome = "Arthur";
Heroi h2 = new Heroi();
h2.nome = "Elena";

2. A Forja do Código: Definição vs Uso

Para dominar a POO, deves entender que um ficheiro define o que o objeto é e outro ficheiro (ou bloco) decide quando o objeto entra em ação.

01 A Classe (O Molde com Funções)

Aqui definimos o Estado (variáveis) e o Comportamento (funções, que em POO chamamos de métodos). Repara que esta classe não tem o método main, pois ela é apenas um modelo.

Heroi.java 
// Definição do molde do aventureiro
class Heroi {
    // 1. Atributos (O que ele TEM)
    String nome;
    int vida;

    // 2. Métodos/Funções (O que ele FAZ)
    void gritarVitoria() {
        System.out.println(nome + " diz: Pela glória do Java!");
    }

    void exibirStatus() {
        System.out.println("--- Status ---");
        System.out.println("Nome: " + nome + " | HP: " + vida);
    }
}

02 O Programa Principal (Consumindo a Classe)

Este é o script que o computador executa. Ele "chama" o molde Heroi, cria instâncias físicas e executa as suas funções.

Principal.java 
public class Principal {
    public static void main(String[] args) {
        
        // Criando o primeiro objeto (Arthur)
        Heroi h1 = new Heroi();
        h1.nome = "Arthur";
        h1.vida = 100;

        // Criando o segundo objeto (Elena)
        Heroi h2 = new Heroi();
        h2.nome = "Elena";
        h2.vida = 85;

        // Chamando as funções que definimos no molde
        h1.exibirStatus();
        h1.gritarVitoria();

        System.out.println(""); // Linha em branco

        h2.exibirStatus();
        h2.gritarVitoria();
    }
}

3. Erros Arcanos: NullPointerException

Esquecer o "new"

No Java, declarar a variável (Heroi h;) apenas cria a etiqueta. Se tentares usar h.nome sem fazer o new Heroi(), o programa morre com erro de valor nulo.

Acesso Fora do Escopo

Atributos definidos numa classe só podem ser acedidos através de um objeto. Não podes escrever apenas nome = "..." dentro do main se o nome pertence à classe Heroi.
