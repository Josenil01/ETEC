1. Pilares do Comportamento

Métodos

São as "funções" que vivem dentro das classes. Definem o que o objeto consegue fazer.

Parâmetros

Os dados que enviamos para dentro do método. São os "ingredientes" para a ação.

Retorno

O valor que o método devolve. O comando return finaliza a tarefa e entrega o resultado.

Void

Usado quando o método executa uma ação (como imprimir algo) mas não devolve nenhum dado.

01 A Classe Mestre (Lógica de Combate e Cura)

Personagem.java 
class Personagem {
    String nome;
    int vida;
    int forca;

    // MÉTODO VOID (Sem retorno): Apenas exibe o status
    void exibirStatus() {
        System.out.println(nome + " [HP: " + vida + "]");
    }

    // MÉTODO COM PARÂMETRO: Recebe valor para reduzir HP
    void receberDano(int valor) {
        vida = vida - valor;
        System.out.println(nome + " sofreu " + valor + " de dano!");
        if (vida <= 0) {
            vida = 0;
            System.out.println("O herói " + nome + " caiu em combate...");
        }
    }

    // MÉTODO COM RETORNO: Calcula dano final e envia resultado
    int atacar() {
        int danoCritico = forca + 10;
        System.out.println(nome + " prepara um golpe poderoso!");
        return danoCritico;
    }

    // MÉTODO BOLEANO: Verifica se ainda pode lutar
    boolean estaVivo() {
        return vida > 0;
    }
}

02 A Arena de Simulação (O Programa Principal)

Arena.java 
public class Arena {
    public static void main(String[] args) {
        
        // Criando o Aventureiro
        Personagem heroi = new Personagem();
        heroi.nome = "Arthur";
        heroi.vida = 100;
        heroi.forca = 25;

        // Criando um Monstro
        Personagem orc = new Personagem();
        orc.nome = "Orc Brutamontes";
        orc.vida = 50;

        System.out.println("--- INÍCIO DO DUELO ---");
        heroi.exibirStatus();
        orc.exibirStatus();

        // Arthur ataca o Orc
        int forcaDoAtaque = heroi.atacar(); // Pega o retorno
        orc.receberDano(forcaDoAtaque);      // Passa como parâmetro

        // Verificação final
        if (!orc.estaVivo()) {
            System.out.println("Vitória de " + heroi.nome + "!");
        }
    }
}

3. Erros Arcanos: Void vs Return

Tentar guardar um "Void"

Se um método é void, ele não devolve nada. Tentar fazer int x = p.exibirStatus() dará erro de compilação.

O Tipo de Retorno Errado

Se declaras int calcular(), deves obrigatoriamente usar return com um número inteiro.
