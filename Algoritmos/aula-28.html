Aprenda a criar hierarquias de poder. Use a Herança para transmitir genes de código e o Polimorfismo para que cada descendente tenha a sua própria alma.

1. Pilares da Linhagem

Herança (Is-A)

Estabelece uma relação de "é um". Ao usar extends, a Subclasse (filho) ganha automaticamente todos os atributos e métodos da Superclasse (pai). É o fim do código repetido.

Poder do Antepassado

A palavra-chave super é o teu cordão umbilical. Permite invocar o construtor ou métodos originais do pai, garantindo que a base do objeto seja construída corretamente antes das customizações.

Sobrescrita Tática

Com @Override, o filho diz: "Eu sei fazer o que o meu pai faz, mas vou fazer do meu jeito". Isto permite redefinir a lógica de um método sem mudar o seu nome ou assinatura.

Polimorfismo

A capacidade de tratar múltiplos tipos de filhos como se fossem o pai genérico. Um exército de Guerreiros, Magos e Arqueiros pode ser comandado como uma simples List<Personagem>.

2. Laboratório: A Árvore Genealógica das Classes

"Todo o herói é um personagem, mas nem todo o personagem é um guerreiro. Vamos criar uma base comum e especializar as lendas."

01 Personagem.java (A Fundação)

Personagem.java 
class Personagem {
    protected String nome; 
    protected int vida;

    // Construtor do Pai
    public Personagem(String nome, int vida) {
        this.nome = nome;
        this.vida = vida;
    }

    public void atacar() {
        System.out.println(nome + " realiza um ataque básico.");
    }
}

02 As Classes Filhas (Especialização e Super)

ClassesEspeciais.java 
class Guerreiro extends Personagem {
    private int armadura;

    public Guerreiro(String nome, int armadura) {
        super(nome, 150); // Chama o construtor do pai com vida base alta
        this.armadura = armadura;
    }

    @Override
    public void atacar() {
        System.out.println(nome + " desfere um golpe de espada esmagador!");
    }
}

class Mago extends Personagem {
    private int mana;

    public Mago(String nome, int mana) {
        super(nome, 80); // Mago tem vida frágil
        this.mana = mana;
    }

    @Override
    public void atacar() {
        System.out.println(nome + " conjura uma bola de fogo explosiva!");
    }
}

class Arqueiro extends Personagem {
    public Arqueiro(String nome) {
        super(nome, 100);
    }

    @Override
    public void atacar() {
        System.out.println(nome + " dispara uma flecha certeira no ponto fraco!");
    }
}

3. Erros Arcanos: Herança Indesejada

Trocar Subclasse por Superclasse

Podes fazer Personagem p = new Guerreiro(), mas NUNCA Guerreiro g = new Personagem(). Um Guerreiro é sempre um Personagem, mas um Personagem genérico pode não ter os genes de um Guerreiro.

Esquecer o super() no Construtor

Se o pai tem um construtor com parâmetros, o filho é OBRIGADO a chamá-lo com super(...) logo na primeira linha do seu construtor. Caso contrário, a linhagem é quebrada.
