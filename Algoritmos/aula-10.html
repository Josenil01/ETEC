1. A Prova do Mestre

Nesta fase da jornada, já não basta saber os comandos isoladamente. Um verdadeiro programador sabe integrar. Programas reais combinam laços de repetição para processar dados, estruturas de decisão para validar entradas e operadores para transformar valores.

O nosso objetivo hoje é construir sistemas que "pensem" do início ao fim, tratando erros do utilizador e entregando resultados precisos.

Estruturas de Repetição: Enquanto (Indeterminado) e Para (Determinado).
Lógica Booleana: Operadores e, ou, nao (Combinação de condições).

3. Laboratório de Integração

01 O Desafio da Arena (Repetição + Decisão + Acumuladores)

Neste programa, lemos o desempenho de vários guerreiros, calculamos a média de poder da equipa e identificamos o guerreiro mais forte.

O Código de Equipa!

Analise como o laço interage com as condições internas.

arena_torneio.por 
programa {
  funcao inicio() {
    inteiro qtd_guerreiros, poder, soma_poder = 0, maior_poder = 0
    real media_poder
    cadeia nome, campeao = ""

    escreva("Quantos guerreiros vão lutar? ")
    leia(qtd_guerreiros)

    para (inteiro i = 1; i <= qtd_guerreiros; i++) {
        escreva("\nNome do guerreiro ", i, ": ")
        leia(nome)
        escreva("Poder (0-100): ")
        leia(poder)

        soma_poder = soma_poder + poder

        se (poder > maior_poder) {
            maior_poder = poder
            campeao = nome
        }
    }
    media_poder = soma_poder / qtd_guerreiros
    escreva("\nMédia da equipa: ", media_poder)
    escreva("\nCampeão: ", campeao, " (", maior_poder, " pts)")
  }
}

02 A Jornada do Herói (Integração Total do Módulo 1)

Este sistema gerencia uma jornada de vários dias. Ele utiliza Entrada de dados, Switch Case para eventos, IF para sobrevivência e um Loop para o tempo.

O Código Mestre!

Este exemplo contém TODOS os conceitos vistos até agora. É a base para qualquer jogo ou sistema complexo.

jornada_mestre.por 
programa {
  funcao inicio() {
    // VARIÁVEIS (Aula 03)
    cadeia nome
    inteiro dias, saude = 100, moedas = 0, acao
    logico vivo = verdadeiro

    escreva("Nome do aventureiro: ")
    leia(nome)
    escreva("Quantos dias dura a missão? ")
    leia(dias)

    // LAÇO PARA (Aula 09) + CONDIÇÃO COMPOSTA (Aula 04/06)
    para (inteiro d = 1; d <= dias e vivo == verdadeiro; d++) {
        escreva("\n--- DIA ", d, " da Jornada ---\n")
        escreva("Saúde: ", saude, " | Moedas: ", moedas, "\n")
        escreva("Escolha: 1-Explorar / 2-Descansar / 3-Treinar: ")
        leia(acao)

        // ESCOLHA-CASO (Aula 07)
        escolha (acao) {
            caso 1: // Explorar (Ganha ouro, perde saúde)
                moedas = moedas + 50
                saude = saude - 30
                escreva("Encontraste tesouros, mas enfrentaste monstros!\n")
                pare
            caso 2: // Descansar (Recupera saúde, gasta ouro)
                saude = saude + 20
                moedas = moedas - 10
                escreva("Dormiste numa taverna segura. Estás revigorado.\n")
                pare
            caso 3: // Treinar (Nada muda, só gasta tempo)
                escreva("Passaste o dia afiando a tua espada.\n")
                pare
            outro_caso:
                escreva("Ficaste confuso e não fizeste nada.\n")
        }

        // ESTRUTURA SE (Aula 06) + OPERADORES RELACIONAIS (Aula 04)
        se (saude <= 0) {
            vivo = falso
        }
        se (saude > 100) { saude = 100 } // Limite de saúde
    }

    // RESULTADO FINAL
    se (vivo) {
        escreva("\nPARABÉNS, ", nome, "! Sobreviveste com ", moedas, " moedas!")
    } senao {
        escreva("\nINFELIZMENTE, ", nome, " pereceu na jornada... Game Over.")
    }
  }
}
