Aula 18: Escudo contra Falhas

O seu código vai falhar. Aprenda a usar o Try/Except para capturar erros e manter o seu programa vivo mesmo sob ataque de dados inválidos.

📚 Resumo Teórico

Erros acontecem. Podem ser erros do utilizador (digitar letras onde deviam ser números) ou erros do sistema (divisão por zero). Sem tratamento, o Python para a execução imediatamente.

O Tratamento de Exceções permite que o código "tente" executar um bloco e, se algo correr mal, ele execute um plano de emergência em vez de fechar.

  • try: O bloco de código que "tentamos" executar.
  • except: O que fazer se um erro específico ocorrer.
  • else: Executado apenas se não houver erros.
  • finally: Executado sempre, independentemente de erros.

1. A Estrutura do Escudo

Esquema Lógico
try:
    # Ação arriscada
except ValueError:
    # Plano de emergência
finally:
    # Limpeza final

🛡️ Prevenção: Capturamos erros específicos para dar mensagens úteis ao utilizador.

🛡️ Continuidade: O programa continua a rodar após o bloco except.

🛡️ Segurança: O finally garante que ficheiros ou conexões sejam fechados sempre.

2. Exemplos Práticos Comentados

01. O Input Teimoso (ValueError)

valida_idade.py 
try:
    # Tentamos converter o texto em número inteiro
    idade = int(input("Quantos anos tem o teu herói? "))
    
    if idade < 0:
        print("Um herói não pode ter idade negativa!")
    else:
        print(f"Nível de experiência ajustado para {idade} anos.")

except ValueError:
    # Captura o erro caso o utilizador digite letras em vez de números
    print("❌ Erro Arcano: Por favor, digita apenas números inteiros!")

02. Partilha de Loot (ZeroDivisionError)

divisao_ouro.py 
try:
    ouro = float(input("Total de ouro encontrado: "))
    membros = int(input("Dividir por quantos heróis na equipa? "))
    
    # Operação aritmética que pode gerar erro
    cada_um = ouro / membros
    
    print(f"Cada aventureiro recebe {cada_um:.2f} moedas de ouro.")

except ZeroDivisionError:
    print("❌ Maldição Matemáticas: Não podes dividir tesouros com o nada (zero pessoas)!")
except ValueError:
    print("❌ Erro de Escrita: Introduz apenas valores numéricos válidos.")

03. Acedendo à Mochila (IndexError)

mochila_segura.py 
mochila = ["Poção", "Espada", "Mapa"]

try:
    print(f"Tens {len(mochila)} itens na mochila.")
    indice = int(input("Qual a posição do item que queres usar? (0 a 2): "))
    
    # Tentar aceder a um índice que pode não existir
    item = mochila[indice]
    print(f"Usaste o item: {item}")

except IndexError:
    print("❌ Espaço Vazio: Esse compartimento da mochila não existe!")
except ValueError:
    print("❌ Erro: Digita o número da posição (ex: 0, 1, 2).")

04. Consulta na Loja (KeyError)

consulta_itens.py 
catalogo = {"espada": 50, "escudo": 30, "arco": 40}

procura = input("Que item desejas comprar? ").lower()

try:
    # Tenta aceder a uma chave no dicionário
    preco = catalogo[procura]
    print(f"O preço do {procura} é {preco} moedas.")
    
except KeyError:
    print(f"❌ Fora de Stock: O item '{procura}' não existe no catálogo.")

3. Laboratório: A Loja Indestrutível

Um sistema de compras que usa um laço while infinito que só termina quando os dados são válidos, protegendo o programa contra falhas.

loja_segura.py 
while True:
    try:
        preco = float(input("Preço do item: "))
        quantidade = int(input("Quantidade: "))
        total = preco * quantidade
        print(f"Total da compra: {total:.2f} moedas.")
        break # Sai do loop se tudo correr bem
    except Exception as erro:
        print(f"⚠️ Algo falhou: {erro}. Tenta novamente.")
    finally:
        print("--- Verificação de Segurança Concluída ---")

4. Erros Arcanos: Escudos Mal Colocados

Except Genérico (Broad Except)

Usar apenas except: sem especificar o erro pode esconder bugs reais que precisas de ver.

except: pass # MÁ PRÁTICA!
# Nunca saberás o que falhou.

Blocos Try Demasiado Grandes

Colocar 50 linhas dentro de um try dificulta saber qual delas causou a exceção.

Dica: Coloca no try apenas a linha que pode falhar.
