Aula 19: Persistência Arcana

Faça o seu código recordar. Aprenda a ler e escrever ficheiros para que os dados do seu herói nunca mais se percam no abismo da memória RAM.

📚 Resumo Teórico

Até agora, as nossas variáveis viviam na Memória RAM — volátil e temporária. Para guardar dados permanentemente, precisamos de usar ficheiros no Disco Rígido.

Em Python, o processo segue sempre três passos: Abrir, Manipular e Fechar.

  • Modo 'w' (Write): Escreve no ficheiro (apaga o que lá estava).
  • Modo 'a' (Append): Adiciona ao final do ficheiro.
  • Modo 'r' (Read): Apenas lê o conteúdo.
  • Context Manager (with): A forma moderna e segura de abrir ficheiros que fecha o "portal" automaticamente para nós.

1. O Encantamento "with"

A Escrita Segura
with open("save.txt", "w") as ficheiro:
    ficheiro.write("Nível 10")

💾 Automatização: O with garante que o ficheiro é fechado mesmo que ocorra um erro.

💾 Segurança: Evita a corrupção de dados por esquecimento do .close().

💾 Limpeza: O código fica mais limpo e legível.

2. Exemplos Práticos Comentados

01. Gravando a Jornada (Append + Loop + Input)

diario_aventureiro.py 
print("--- DIÁRIO DE MISSÕES ---")

# Usamos um laço while (Aula 13) para múltiplas entradas
while True:
    entrada = input("O que conquistaste hoje? (ou 'fim' para salvar): ")
    
    if entrada.lower() == 'fim':
        break
    
    # 'a' (append) para adicionar sem apagar o histórico anterior
    with open("historico.txt", "a", encoding="utf-8") as arquivo:
        arquivo.write(f"- {entrada}\n")

print("Pergaminho atualizado com sucesso!")

02. Consultando Memórias (Read + Try/Except + Loop)

reler_jornada.py 
try:
    with open("historico.txt", "r", encoding="utf-8") as arquivo:
        print("--- RELEMBRANDO A TUA HISTÓRIA ---")
        
        # Lendo linha por linha com um laço for
        for indice, linha in enumerate(arquivo, 1):
            print(f"Dia {indice}: {linha.strip()}")

except FileNotFoundError:
    # Tratamento de erro (Aula 18)
    print("❌ Erro: Ainda não escreveste nada no teu diário.")

4. Erros Arcanos: Ficheiros Corrompidos

FileNotFoundError

O erro mais comum. Tentaste ler um ficheiro que não existe no caminho indicado.

Solução: Usar Try/Except ou verificar se o caminho está correto.

UnicodeDecodeError

Acontece quando o ficheiro tem acentos e não defines o encoding="utf-8".

Dica: Usa sempre utf-8 para compatibilidade global.
