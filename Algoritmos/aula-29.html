Prepare-se para o erro. Aprenda a capturar as Exceptions do Java para que os seus sistemas sobrevivam até aos utilizadores mais destrutivos.

1. Pilares da Defesa

Try (Tentar)

Envolve o código "perigoso" que pode falhar. É a zona protegida onde o Java fica alerta para qualquer anomalia.

Catch (Capturar)

O plano de emergência. Se o try falhar, o catch entra em ação para tratar o erro e evitar o crash do sistema.

Finally

O código que corre sempre. Ideal para fechar conexões ou limpar a memória, quer tenha ocorrido um erro ou não.

Exception

A superclasse de todos os erros. No Java, um erro é um objeto que contém detalhes sobre o que falhou e onde.

2. Laboratório: O Sistema de Arena Resiliente

"Um mestre não teme o erro, ele prepara o código para lidar com ele. Vamos proteger a partilha de tesouros e o cálculo de poder."

01 CalculadoraArcana.java (A Classe Perigosa)

CalculadoraArcana.java 
class CalculadoraArcana {
    // Este método pode falhar se 'herois' for zero
    public double dividirTesouro(double ouro, int herois) {
        return ouro / herois;
    }

    // Pode falhar se o índice não existir no array
    public String obterItem(String[] bau, int indice) {
        return bau[indice];
    }
}

02 ArenaSegura.java (O Executor Protegido)

ArenaSegura.java 
import java.util.Scanner;

public class ArenaSegura {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        CalculadoraArcana calc = new CalculadoraArcana();

        try {
            System.out.print("Quantas moedas de ouro encontraste? ");
            double ouro = scanner.nextDouble();

            System.out.print("Dividir por quantos heróis? ");
            int qtd = scanner.nextInt();

            double cadaUm = calc.dividirTesouro(ouro, qtd);
            System.out.println("Cada herói recebe: " + cadaUm);

        } catch (java.util.InputMismatchException e) {
            System.out.println("⚠️ Erro: Precisas de digitar números, não runas (letras)!");
        } catch (ArithmeticException e) {
            System.out.println("⚠️ Erro: Não podes dividir ouro por zero heróis!");
        } finally {
            System.out.println("🧹 Sistema de segurança finalizado. Scanner fechado.");
            scanner.close();
        }
    }
}

3. Erros Arcanos: O Escudo de Papel

Engolir a Exceção (Silent Fail)

Fazer um catch com o corpo vazio é um crime. Se o código falhar e não avisares nada, nunca saberás porque é que o teu ouro desapareceu!

Ordem dos Catchs

Captura sempre o erro mais específico primeiro. Se colocares o Exception genérico no topo, ele capturará tudo e os catchs específicos debaixo nunca correrão.
