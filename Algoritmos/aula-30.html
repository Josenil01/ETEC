Aula 30: O Arquiteto de Sistemas

Chegou o momento da verdade. Iremos forjar um Sistema de Gestão de Inventário completo, unindo todos os pilares da Orientação a Objetos.

1. Pilares da Integração

POO

Estrutura Social

Uso de Herança para itens comuns e Polimorfismo para que cada item "se use" de forma diferente.

Cofre de Dados

Atributos privados e validações em Setters para impedir que um item tenha preço ou peso negativo.

Mochila Dinâmica

Uso de ArrayList<Item> para gerir o crescimento do inventário em tempo real.

!

Resiliência

Proteção contra InputMismatch e índices inválidos com blocos try-catch.

2. A Missão: Sistema de Inventário

"Um herói não é nada sem o seu equipamento. Vais construir o motor que gere as armas, as armaduras e as poções da Guilda."

Especificações Técnicas:

  • Classe Pai Item: Atributos privados (nome, preço) com Getters/Setters.
  • Subclasses Arma e Pocao: Devem herdar de Item e ter atributos específicos.
  • Polimorfismo: O método usarItem() deve imprimir mensagens diferentes para cada tipo.
  • Gestor: Uma classe que contenha o ArrayList de itens.

Fluxo do Programa:

  • Menu Loop: (1) Adicionar Item, (2) Listar Inventário, (3) Usar Item, (4) Sair.
  • Segurança: Impedir que o programa feche caso o utilizador digite letras no menu de números.

3. Solução do Mestre

01 A Base e Linhagem (Item.java e Extensões)

Hierarquia.java 
// CLASSE PAI COM ENCAPSULAMENTO
class Item {
    private String nome;
    private double preco;

    public Item(String nome, double preco) {
        this.nome = nome;
        this.preco = preco;
    }

    public String getNome() { return nome; }
    
    // MÉTODO POLIMÓRFICO
    public void usar() {
        System.out.println("Usando item genérico...");
    }
}

// SUBCLASSE ARMA
class Arma extends Item {
    private int dano;

    public Arma(String nome, double preco, int dano) {
        super(nome, preco);
        this.dano = dano;
    }

    @Override
    public void usar() {
        System.out.println("Atacando com " + getNome() + "! Causaste " + dano + " de dano.");
    }
}

// SUBCLASSE POÇÃO
class Pocao extends Item {
    private int cura;

    public Pocao(String nome, double preco, int cura) {
        super(nome, preco);
        this.cura = cura;
    }

    @Override
    public void usar() {
        System.out.println("Bebendo " + getNome() + "! Recuperaste " + cura + " HP.");
    }
}

02 O Motor do Jogo (Principal.java com ArrayList e Try-Catch)

Principal.java 
import java.util.ArrayList;
import java.util.Scanner;

public class Principal {
    public static void main(String[] args) {
        Scanner leitor = new Scanner(System.in);
        ArrayList<Item> inventario = new ArrayList<>();
        int opcao = 0;

        while (opcao != 4) {
            try {
                System.out.println("\n--- INVENTÁRIO DA GUILDA ---");
                System.out.println("1. Adicionar Espada | 2. Listar Itens | 3. Usar Item | 4. Sair");
                System.out.print("Escolha: ");
                opcao = Integer.parseInt(leitor.nextLine()); // Evita erros de buffer

                switch (opcao) {
                    case 1:
                        inventario.add(new Arma("Espada Larga", 150.0, 25));
                        System.out.println("✅ Arma adicionada!");
                        break;
                    case 2:
                        for (Item i : inventario) {
                            System.out.println("- " + i.getNome());
                        }
                        break;
                    case 3:
                        if (inventario.isEmpty()) throw new Exception("Mochila vazia!");
                        // Polimorfismo: o .usar() sabe se é arma ou poção!
                        inventario.get(0).usar();
                        break;
                }
            } catch (Exception e) {
                System.out.println("❌ Erro: " + e.getMessage());
            }
        }
        leitor.close();
    }
}
