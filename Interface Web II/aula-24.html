Aula 24: A Fábrica de Clones

Bem-vindo a Kamino. Abandone o código repetitivo e descubra o Paradigma Orientado a Objetos (POO): Classes, Construtores, Instâncias e Encapsulamento.

1. O Problema da Repetição

Imagine que recebeu uma encomenda do Mestre Jedi Sifo-Dyas: criar um exército com 200.000 unidades. Se fosse usar o JavaScript clássico (sem POO), teria de criar 200.000 variáveis separadas, cada uma com o seu próprio nome, armadura e armas. Seria um pesadelo de manutenção.

A solução dos Kaminoanos (e dos Arquitetos de Software) é a Programação Orientada a Objetos (POO). Em vez de construir cada clone do zero, criamos um "Molde de DNA" (uma Classe). A partir desse único molde genético, usamos uma máquina de fabrico para gerar milhões de Clones independentes (os Objetos ou Instâncias).

🔮 Olhando para o Futuro: A POO não fica por aqui. Mais tarde, aprenderemos que os clones podem sofrer mutações avançadas usando Herança (ex: class ARCTrooper extends CloneTrooper), herdando tudo da classe base e ganhando novas habilidades exclusivas!

2. A Tecnologia de Clonagem (Sintaxe)

O Molde de DNA (A Classe)

A Classe é a planta de arquitetura. Define o que o clone tem (Atributos) e o que ele faz (Métodos).

class Clone {
  // O Tubo de Criação:
  constructor(id, funcao) {
    this.designacao = id;
    this.patente = funcao;
  }

  // Ação do Clone:
  atirar() {
    return 'Pew pew!';
  }
}
A Instanciação (O Objeto)

A palavra new é o botão que liga as máquinas de Kamino. Ela pega no molde da Classe e cospe um Objeto físico para a memória RAM.

// Criando o Capitão Rex
const rex = new Clone('CT-7567', 'Capitão');

// Criando o Comandante Cody
const cody = new Clone('CC-2224', 'Comandante');

console.log(rex.designacao); // CT-7567
cody.atirar(); // Pew pew!
A Regra de Ouro: O que é o "this" e o Encapsulamento?

A palavra this significa literalmente "isto" (ou "eu mesmo"). Dentro da Classe, o molde não sabe qual será o nome do clone no futuro. Quando escrevemos this.designacao = id, estamos a dizer: "A designação Deste Clone Específico será igual ao ID passado." É o this que impede a armadura do Cody de se misturar com a do Rex!

O Encapsulamento: Nunca devemos mudar as propriedades de um clone por fora (ex: rex.patente = "General"). Em vez disso, a própria Classe deve ter um "Método Mutador" (ex: promover()) que altera o seu estado interno de forma segura e controlada.

A Planta (Blueprint) 2. Criando a Classe Clone

No JavaScript, criamos a classe. Repare na adição do método promover(), que demonstra como o objeto altera a sua própria patente de forma segura e encapsulada.

MoldeGenetico.js
class CloneTrooper {
    
    // O CONSTRUTOR: Roda automaticamente no momento do "new"
    constructor(idRecebido, patenteRecebida) {
        // Atributos (Propriedades / DNA)
        this.designacao = idRecebido;
        this.patente = patenteRecebida;
        this.lealdade = "República Galáctica"; // Valor fixo para todos
    }

    // MÉTODOS: O que os clones conseguem fazer (Ações)
    atirar() {
        return `<strong>${this.designacao}</strong> disparou o seu blaster DC-15A! Pew! Pew!`;
    }

    apresentar() {
        return `Senhor! Sou o <strong>${this.patente} ${this.designacao}</strong>, a aguardar ordens!`;
    }
    
    // MÉTODO MUTADOR: Altera o estado interno do objeto de forma segura
    promover() {
        this.patente = "Comandante";
        return `O clone <strong>${this.designacao}</strong> foi promovido a Comandante!`;
    }
}

Instanciação (DOM) 4. A Fábrica Defensiva

Quando o botão é clicado, adicionamos a **Validação Defensiva** (Aula 16) para garantir que não geramos clones vazios. Para além disso, aplicamos a Regra da Unicidade (como as Chaves Primárias no PostgreSQL) para garantir que não existem dois clones com o mesmo ID. Só depois executamos a magia da palavra new e guardamos o clone na memória.

fabrica.js
const form = document.getElementById('kamino-form');
const grid = document.getElementById('exercito-grid');

// Array para guardar todas as "Instâncias" geradas
const exercito = [];

form.addEventListener('submit', (e) => {
    e.preventDefault();
    
    const idDigitado = document.getElementById('clone-id').value.trim();
    const patenteSelecionada = document.getElementById('clone-role').value;

    // Defesa Nível 1: Evitar a criação de Clones Defeituosos (Sem ID)
    if (!idDigitado) {
        alert('Atenção: Insira um ID válido para iniciar a clonagem!');
        return;
    }

    // Defesa Nível 2: Prevenir Clones Duplicados (Regra da Unicidade)
    const cloneJaExiste = exercito.some(clone => clone.designacao === idDigitado);
    if (cloneJaExiste) {
        alert('Falha na Clonagem: Já existe uma unidade com esta designação na base de dados!');
        return;
    }

    // A MAGIA DA POO: Nasce um novo Objeto físico baseado na Planta!
    const novoSoldado = new CloneTrooper(idDigitado, patenteSelecionada);
    
    exercito.push(novoSoldado); // Guarda na memória
    renderizarClone(novoSoldado); // Desenha na tela
    
    form.reset();
});

Renderização Dinâmica 5. Dando Vida Aos Botões

Na função que desenha o HTML do soldado, ligamos os botões da interface aos Métodos da Classe que definimos, incluindo o botão de "Promover" que irá mutar o objeto interno e atualizar o DOM.

fabrica.js (Parte 2)
function renderizarClone(objetoClone) {
    const card = document.createElement('div');
    card.className = 'clone-card';

    card.innerHTML = `
        <div class="clone-header">
            <span class="clone-id">${objetoClone.designacao}</span>
            <span class="clone-role">${objetoClone.patente}</span>
        </div>
        <div class="flex gap-2 mt-2">
            <button class="clone-action-btn flex-1 btn-atirar">Atirar</button>
            <button class="clone-action-btn flex-1 btn-relato">Relato</button>
            <button class="clone-action-btn flex-1 btn-promover">Promover</button>
        </div>
    `;

    // 1. Ligar Método Atirar
    card.querySelector('.btn-atirar').addEventListener('click', () => {
        mostrarNoLog(objetoClone.atirar());
    });

    // 2. Ligar Método Mutador (Promover)
    card.querySelector('.btn-promover').addEventListener('click', () => {
        // O Objeto altera-se a si mesmo pelas regras da Classe!
        mostrarNoLog(objetoClone.promover());
        
        // Atualizamos o HTML visual (DOM) para refletir o novo estado do Objeto
        card.querySelector('.clone-role').innerText = objetoClone.patente;
        card.style.borderColor = '#eab308'; // Borda Dourada para Comandantes
    });

    grid.appendChild(card);
}
