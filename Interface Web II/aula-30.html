Aula 30: A Vitrine Modular

O Prólogo para o React. Aplique os conceitos de Arquitetura Limpa ao seu Front-End construindo Componentes e "Blocos de LEGO" visuais.

1. A Ilusão do Front-End Limpo

Nas Aulas 28 e 29, provámos o quão desastroso é misturar SQL com Rotas no Back-End, e resolvemos isso com Controllers e Repositories. Agora, vamos ser sinceros e olhar para o nosso Front-End (cliente.js).

O nosso cliente.js faz pedidos à rede (fetch), bloqueia botões, gere o submit do formulário e, esmagado no meio de tudo isto... injeta pedaços gigantescos de código HTML com crases (` `).

E se o cartão da relíquia precisar de uma imagem? E de um botão "Editar"? E de um botão "Apagar"? O ficheiro cliente.js vai transformar-se num novo monstro incontrolável. Para resolver isto, vamos introduzir a Componentização: a arte de transformar a Interface de Utilizador (UI) em pequenos blocos de LEGO (Componentes) independentes.

2. A Fábrica de LEGO Visual

Um Componente é simplesmente uma Função de JavaScript pura. O seu único trabalho no mundo é receber Dados (que vamos chamar de props) e devolver a estrutura HTML desse objeto formatada.

Processo de Componentização Front-End

cliente.js (O Caos Visual)

const reliquias = await fetch('...');

reliquias.forEach(rel => {

shelf.innerHTML += `
  <div class="relic-card">
    <span>${rel.id}</span>

  </div>`

⬇
components/ReliquiaCard.js export const ReliquiaCard = (props) => {

}
cliente.js import { ReliquiaCard } from './components...';

reliquias.forEach(rel => {
  shelf.insertAdjacentHTML('beforeend', ReliquiaCard(rel));

/index.html 1. A Evolução do Navegador (type="module")

Para o navegador suportar o sistema moderno de ficheiros (onde um ficheiro JS importa outro usando import e export), precisamos de fazer uma alteração vital na forma como carregamos o nosso script no final do index.html.

index.html 
<!-- Antes: -->
<!-- <script src="cliente.js"></script> -->

<!-- AGORA: Declaramos que o nosso JS usa a arquitetura de Módulos (ES Modules)! -->
<script type="module" src="cliente.js"></script>
</body>

/components 2. A Fábrica (ReliquiaCard.js)

Retiramos toda a poluição visual do nosso gestor de rede. Esta função não sabe o que é uma API ou um Formulário. O seu único objetivo de vida é receber dados (chamados convencionalmente de props) e devolver o HTML lindamente montado. Nota: As classes CSS (.relic-card, etc.) continuam guardadas no seu ficheiro global style.css!

ReliquiaCard.js 
// Usamos 'export' para permitir que o cliente.js importe esta função
export const ReliquiaCard = (props) => {
    
    // Devolvemos a String de HTML encapsulada
    return `
        <div class="relic-card">
            <span class="relic-id">#${props.id}</span>
            <div class="relic-title">${props.nome}</div>
            <div class="text-xs text-amber-700 uppercase tracking-widest">${props.epoca}</div>
            
            <!-- Olhe como é fácil adicionar novos botões sem sujar o cliente.js! -->
            <div class="mt-4 flex gap-2 justify-center">
                <button class="text-xs bg-slate-200 px-2 py-1 rounded hover:bg-slate-300">Editar</button>
                <button class="text-xs bg-red-100 text-red-600 px-2 py-1 rounded hover:bg-red-200">Destruir</button>
            </div>
        </div>
    `;
};

/ 3. A Montagem Final (cliente.js limpo)

Observe como o nosso cliente.js recupera a sua elegância. Agora ele apenas importa a "peça de LEGO" do ficheiro externo e passa-lhe os dados da rede!

cliente.js 
// Importamos a nossa "Fábrica" de UI no topo do ficheiro
import { ReliquiaCard } from './components/ReliquiaCard.js';

const API_URL = 'http://localhost:3000/api/reliquias';
const shelf = document.getElementById('museum-shelf');

async function carregarPrateleira() {
    try {
        const resposta = await fetch(API_URL);
        const reliquias = await resposta.json();
        
        shelf.innerHTML = ''; 

        if (reliquias.length === 0) {
            shelf.innerHTML = '<p>O museu está vazio.</p>';
            return;
        }

        // O SEGREDO DO REACT: Transformamos a lista de dados numa lista de componentes e injetamos tudo de uma só vez!
        shelf.innerHTML = reliquias.map(rel => ReliquiaCard(rel)).join('');

    } catch (erro) {
        console.error('Erro ao carregar:', erro);
    }
}

// ... Código do POST continua idêntico, mas usando ReliquiaCard(novaReliquia)

O Limite Humano (A Dor do Vanilla JS)

Criámos um sistema lindíssimo de componentes com Vanilla JavaScript (JS Puro). No entanto, deparámo-nos com uma parede inultrapassável de performance e manutenção.

Se quisermos atualizar dinamicamente o nome de apenas uma relíquia na prateleira... o nosso código atual é tão burro que teria de usar innerHTML = '' e re-desenhar TODOS os cartões do zero na tela, o que causa lentidão. A única alternativa manual seria selecionar o ID do cartão, encontrar o nó do texto e manipulá-lo com document.querySelector. Imagine fazer isso para 100 relíquias!

Bem-vindo ao momento pelo qual ansiava. Está pronto para o React.js.

4. Expedições Visuais

Nível: Escultor Aprendiz (Fácil)

01

Micro-Componentes: Assim como a Relíquia é um componente, os botões dentro dela também poderiam ser! Crie uma constante global chamada BotaoBase = (texto) => { return \`<button class="btn...">${texto}</button>\`; }. Use essa constante dentro do ReliquiaCard.js para gerar o botão "Editar" e "Destruir" sem repetir HTML!

Nível: O Fim da Linha (Desafio Filosófico)

02

A Limitação de Estado (Pesquisa):

Seja honesto com a arquitetura: no nosso componente ReliquiaCard (Passo 2), o que acontece se quisermos mudar a cor do botão "Editar" para verde quando o rato passa por cima (hover em JS, não em CSS)?

Como o nosso componente retorna uma simples String estática (texto puro), nós não conseguimos anexar um addEventListener facilmente dentro da função.

A Missão: Pesquise sobre "O que é o DOM Virtual do React". Tente entender como é que uma biblioteca externa como o React resolve esta "burrice" das Strings de HTML e permite que os componentes tenham vida, memória e eventos! (Este é o seu trabalho de casa final antes de entrar no próximo nível).
