Aula 21: O Baú do Mundo

Abra o inventário. Aprenda a operação de Leitura (Read) usando o comando SELECT e preencha a sua interface dinamicamente.

1. A Leitura dos Dados (CRUD)

Na aula passada aprendemos a Spawnar blocos usando a operação Create. Mas de que adianta guardar um item se não pudermos olhar para ele depois? Voltando à nossa sigla CRUD:

C Create (Criar)
R Read (Ler)
U Update (Atualizar)
D Delete (Apagar)

Hoje o foco é o Read (Ler). Vamos pedir ao nosso servidor Node.js que vá ao banco de dados Neon.tech, abra a tabela inventario e devolva os blocos criados. Em seguida, usaremos laços de repetição (Aula 07) para desenhar o baú no ecrã.

2. O Feitiço de Visão (SELECT) e Seus Limites

Procurando na Base de Dados

O comando mais utilizado de todo o SQL é o SELECT. Ele diz ao banco de dados quais colunas queremos ver e de que tabela.

-- Traz TODAS as colunas (*) de TODOS os blocos
SELECT * FROM inventario;

-- Traz apenas itens específicos (Filtro WHERE)
SELECT material, quantidade FROM inventario WHERE material = 'Diamante';

-- BOAS PRÁTICAS: O Limite do Baú (LIMIT)
-- Traz apenas os últimos 27 itens. Evita crashar o servidor!
SELECT * FROM inventario ORDER BY id DESC LIMIT 27;
O Retorno (A Lista)

O pacote pg converte as linhas do banco num Array JS (ex: [{id: 1, material: 'Terra'}, ...]). Nós enviaremos esse Array ao Front-End em formato JSON.

E se o Baú estiver vazio?

O servidor não dá erro. Ele simplesmente devolve um Array vazio []. É dever do programador Front-End tratar esse "Empty State" avisando o utilizador.

Back-End (Node) 3. A Rota de Leitura (servidor.js)

No nosso servidor, criamos uma nova rota app.get (pois estamos a pedir dados). Nela, executamos o SELECT com o limite de segurança de 27 slots e devolvemos o Array em JSON.

servidor.js
// Rota GET para consultar o inventário
app.get('/api/inventario', async (req, res) => {
    try {
        // 1. A Query: Todos os itens, ordem decrescente, máximo 27 blocos
        const sql = 'SELECT * FROM inventario ORDER BY id DESC LIMIT 27';
        
        // 2. O Banco de Dados executa a query
        const resultado = await pool.query(sql);

        // 3. resultado.rows contém a nossa Lista (Array) de itens!
        res.status(200).json(resultado.rows);

    } catch (erro) {
        console.error(erro);
        res.status(500).json({ erro: 'Falha ao abrir o baú no banco de dados.' });
    }
});

JS Cliente 4. Proteção, Fetch e Renderização!

No cliente.js, implementamos o ciclo completo de assincronia de produção: Ligar o Loader -> Tentar fazer o Fetch (try) -> Desligar o Loader (finally) e usar o forEach para desenhar a grelha.

cliente.js
const btnAbrir = document.getElementById('btn-abrir-bau');
const bauGrid = document.getElementById('bau-grid');
const loader = document.getElementById('loader-bau');

btnAbrir.addEventListener('click', async () => {
    try {
        // 1. Ligar o Loader e preparar o terreno
        loader.classList.remove('hidden');
        bauGrid.innerHTML = '';
        btnAbrir.disabled = true;

        // 2. Pedir os dados ao servidor (GET)
        const res = await fetch('http://localhost:3000/api/inventario');
        
        // O Escudo: Se o servidor der erro 500, atiramos para o catch!
        if (!res.ok) throw new Error('Servidor offline ou erro na API.');
        
        const itensDoBanco = await res.json();
        
        // 3. Tratamento de Baú Vazio (Empty State)
        if (itensDoBanco.length === 0) {
            bauGrid.innerHTML = '<p style="grid-column: span 5; text-align: center; color: #555;">O baú está vazio.</p>';
            return;
        }

        // 4. Laço Mágico: Para cada item, criamos um slot no HTML
        itensDoBanco.forEach((item) => {
            const slot = document.createElement('div');
            slot.className = 'mc-slot';
            
            slot.innerHTML = `
                <span class="mc-item-icon">📦</span>
                <span class="mc-item-qtd">${item.quantidade}</span>
                <div class="mc-item-tooltip">${item.material} (ID: ${item.id})</div>
            `;
            
            bauGrid.appendChild(slot);
        });

    } catch (erro) {
        // Cai aqui se o servidor estoirar ou a net falhar
        bauGrid.innerHTML = `<p style="grid-column: span 5; color: red;">Erro: ${erro.message}</p>`;
    } finally {
        // 5. Finalmente, desligamos o loader em todos os casos
        loader.classList.add('hidden');
        btnAbrir.disabled = false;
        btnAbrir.innerText = 'Baú Aberto (Recarregar)';
    }
});

4. Desafios do Explorador

01
A Lente de Aumento (SQL WHERE):

Imagine que o seu baú tem 1000 itens, mas você só quer puxar a "Madeira" limitada a 5 blocos. Escreva no seu código (ou no painel da Neon) a query SQL para: Selecionar tudo DA tabela inventario ONDE o material for igual a 'Madeira' LIMITADO a 5 itens.

02
Desenhando um Baú Vazio Customizado:

O seu código do Passo 3 já trata o baú vazio injetando um texto cinzento ("O baú está vazio"). Mude a lógica dentro do if (itensDoBanco.length === 0) para desenhar uma <div class="mc-slot"> que mostre apenas um emoji de teia de aranha 🕸️, sinalizando que não há loot nenhum!

03
Desafio Épico: A Barra de Pesquisa Integrada

Na tela do utilizador, crie um campo de input <input type="text" id="busca-input"> e um botão "Pesquisar". A ideia é que o utilizador procure por itens específicos sem precisar de abrir o baú inteiro.

Passos no Back-End (servidor.js):

  • Crie uma nova rota: app.post('/api/pesquisar', async (req, res) => {...})
  • Receba a palavra buscada: const { termo } = req.body;
  • Use o comando SQL com o operador ILIKE (pesquisa aproximada e insensível a maiúsculas):
    SELECT * FROM inventario WHERE material ILIKE $1
  • Execute a query passando o parâmetro com coringas: pool.query(sql, [`%${termo}%`])
  • Devolva o resultado.rows.

Crie o fetch correspondente no Front-End para consumir esta rota e desenhar os itens filtrados na grelha!
