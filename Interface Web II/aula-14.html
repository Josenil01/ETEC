Aula 14: O Lado Sombrio da Força

Abandone a luz do Navegador. Compreenda a arquitetura em camadas e descubra como executar scripts poderosos do Lado do Servidor.

1. A Estrela da Morte (O Servidor)

Até agora, todo o nosso JavaScript correu no Navegador (Front-End). O Navegador é como a Aliança Rebelde: rápido, interativo, mas os seus ficheiros estão expostos a qualquer um que inspecione a página.

Para processar regras de negócio críticas e aceder à Base de Dados (Os Arquivos Imperiais), precisamos de um ambiente seguro, longe dos utilizadores. Esse ambiente é o Servidor Web (Back-End). Tecnologias como o Node.js permitem-nos usar o JavaScript no Lado Sombrio (Servidor), criando a verdadeira arquitetura de aplicações Web em camadas.

2. A Arquitetura em Camadas

1. Cliente / Navegador

A nave do utilizador. Responsável por mostrar a interface gráfica e capturar cliques.

Tecnologias:

HTML, CSS, JS (React, Vue)

2. Servidor Web (App)

A base central. Recebe os pedidos via HTTP, processa a lógica de negócio e toma decisões de segurança.

Tecnologias:

Node.js (Express), Python, PHP

3. Base de Dados

Os arquivos centrais. A memória permanente do sistema, onde dados são lidos e guardados com segurança.

Tecnologias:

PostgreSQL (Neon.tech)

3. O Estaleiro Imperial (Setup Local)

Antes de ligar os motores do Servidor, precisamos instalar as peças fundamentais no nosso computador. Abra o terminal (linha de comandos) na pasta do seu projeto e execute as seguintes ordens:

npm init -y
O Manifesto (Inicializar Projeto) Cria o ficheiro package.json. É o registo oficial do projeto, onde ficam guardadas as informações de todas as bibliotecas e peças que a nossa nave vai usar.
npm install express
O Motor de Dobra (Express.js) Instala o framework Express. É a ferramenta mais famosa do Node.js para criar servidores Web, pois facilita incrivelmente a criação das "rotas" (ex: /api/procurar) que vão ouvir os pedidos HTTP da Rebelião (Cliente).
npm install cors
Os Códigos de Acesso (CORS) Instala o CORS (Cross-Origin Resource Sharing). Sem isto, o Navegador (que corre na porta 5500 do Live Server) bloqueia o Front-End de comunicar com o nosso servidor (na porta 3000) por segurança. O CORS atua como guarda de fronteira e autoriza essa partilha local.

* Aviso Imperial: Para que o comando "npm" funcione, é estritamente obrigatório ter o Node.js instalado no seu sistema operativo.

Camada 1 (Front-End) 1. No ficheiro cliente.js (O Navegador)

Em vez de apontar a transmissão para um domínio na internet (http://servidor-imperial.com/...), você aponta para a sua própria máquina (localhost) na "porta" específica onde o servidor está a ouvir (por exemplo, a porta 3000). O seu fetch ficaria assim:

cliente.js
const form = document.getElementById('sith-form');
const output = document.getElementById('server-output');

form.addEventListener('submit', async (event) => {
    event.preventDefault(); // Impede o recarregamento

    const alvo = document.getElementById('input-alvo').value;
    output.innerHTML = '<div class="loader"></div> Conectando ao Servidor Local...';

    // O Cliente chama o Servidor na própria máquina (localhost) na porta 3000
    const resposta = await fetch('http://localhost:3000/api/procurar', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ nome: alvo })
    });

    const dados = await resposta.json();
    output.innerHTML = `> STATUS: <strong>${dados.mensagem}</strong>`;
});

Camada 2 (Back-End) 2. No ficheiro servidor.js (O Node.js)

No código do servidor, não muda quase nada na rota, mas precisamos de adicionar um Guarda de Fronteira chamado CORS (Cross-Origin Resource Sharing).

Porquê? Por questões de segurança, os navegadores bloqueiam um site (que está a correr no Live Server, por exemplo, na porta 5500) de fazer pedidos para um servidor noutra porta (a porta 3000). Para permitir essa conversa local, o servidor Node.js tem de usar a biblioteca cors.

O código do servidor local seria assim:

servidor.js (Ficheiro Node.js/Express)
const express = require('express');
const cors = require('cors'); // 1. Importar o CORS

const app = express();

// 2. Avisar o servidor para permitir chamadas locais (ignorar o bloqueio do navegador)
app.use(cors()); 
app.use(express.json());

// 3. A nossa rota continua igual!
app.post('/api/procurar', (req, res) => {
    const nomeRecebido = req.body.nome.toLowerCase();
    
    // ... lógica de negócio Imperial aqui ...

    res.json({ mensagem: 'Resposta do meu servidor local!' });
});

// 4. O servidor fica à escuta na porta 3000 do seu computador
app.listen(3000, () => console.log('Servidor local online em http://localhost:3000'));
