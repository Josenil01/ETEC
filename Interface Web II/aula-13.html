Aula 13: O Carteiro da Web

Descodifique a Matrix. Aprenda como a arquitetura Cliente/Servidor funciona, compreenda os métodos HTTP (GET/POST) e o que são QueryStrings.

1. A Comunicação na Matrix

Nas aulas passadas dominámos o Navegador (Front-End). Mas o navegador é apenas uma "janela". Para as coisas serem guardadas de verdade, precisamos de enviar os dados para o Servidor Web (Back-End).

A linguagem que o Navegador (Cliente) e o Servidor usam para conversar chama-se HTTP (Hypertext Transfer Protocol). Sempre que clica num link ou envia um formulário, o Navegador está a despachar uma "carta" com instruções precisas. Se não entender as regras de envio, as suas mensagens secretas poderão ser intercetadas pelos Agentes.

2. Os Protocolos de Ligação

A Forma de Envio (Métodos HTTP)
Método GET (Obter)

Como um postal. A informação vai escrita do lado de fora da carta (na URL). É usado para pedir dados (ex: pesquisar um produto). Nunca use para palavras-passe!

Método POST (Enviar)

Como uma encomenda selada. Os dados vão escondidos dentro do "Corpo" (Body) do pedido. Usado para enviar dados sensíveis ou grandes (ex: registar utilizador).

O Endereço (URL)
https://api.matrix.com/sync?agente=Neo&status=ok

Esta é uma URL com uma QueryString.

  • O ponto de interrogação (?) indica o início dos dados.
  • agente=Neo é um par Chave=Valor.
  • O símbolo & une vários parâmetros diferentes.

Passo 4 Construindo a QueryString (GET)

No método GET, anexamos as variáveis diretamente na URL. Usamos o URLSearchParams para montar isso facilmente.

matrix.js (Parte 2)
btnGet.addEventListener('click', () => {
    // Captura os valores
    const agente = inputAgente.value;
    const senha = inputSenha.value;

    // URLSearchParams formata automaticamente: ?chave=valor&chave2=valor2
    const parametros = new URLSearchParams({
        agente: agente,
        codigo: senha
    });

    // A URL final fica exposta!
    const urlFinal = `${URL_SERVIDOR}?${parametros.toString()}`;

    saida.innerHTML = `
        > [ALERTA DE SEGURANÇA]<br>
        > Efetuando pedido GET...<br>
        > URL Acedida: <span style="color: #ef4444">${urlFinal}</span><br><br>
        > Os Agentes conseguem ver a sua palavra-passe na morada da web!
    `;
});

Passo 5 Criando um Payload (POST)

No método POST, a URL fica "limpa". Os dados vão empacotados num objeto JSON dentro do corpo (Body) do pedido HTTP.

matrix.js (Parte 3)
btnPost.addEventListener('click', () => {
    const agente = inputAgente.value;
    const senha = inputSenha.value;

    // Preparamos os dados como um Objeto
    const corpoMensagem = {
        agente: agente,
        codigo: senha
    };

    // JSON.stringify() converte o Objeto em texto para viajar na rede
    const jsonString = JSON.stringify(corpoMensagem);

    saida.innerHTML = `
        > [CONEXÃO SEGURA]<br>
        > Efetuando pedido POST...<br>
        > URL Acedida: <span style="color: #34d399">${URL_SERVIDOR}</span><br>
        > Corpo do Pedido (Payload Oculto): <br>
        > <span style="color: #34d399">${jsonString}</span>
    `;
});

4. Treino na Sala de Simulação

Nível: Cobaia (Fácil)

01

Montador de Queries: Crie um objeto em JS chamado pesquisa com as propriedades { filme: "Matrix", ano: 1999 }. Use a ferramenta new URLSearchParams(pesquisa).toString() e faça um console.log() do resultado. Repare como o sinal & é adicionado automaticamente!

Nível: Operador (Médio)

02

A Magia do FormData: Em formulários gigantes, capturar o .value um a um é cansativo. Altere o HTML adicionando o atributo name="agente" no primeiro input. No evento de clique do POST, crie const form = document.getElementById('cyber-form'); e depois const dados = new FormData(form);. Imprima dados.get('agente') na consola. O FormData captura todos os inputs que possuam um atributo name!

Nível: O Escolhido (Desafio)

03

O Envio Real (Fetch com POST):

Na Aula 12, fizemos um fetch que implicitamente era um GET. Vamos fazer um POST real para a API pública JSONPlaceholder!
1. Crie uma função async para o clique.
2. Crie um objeto const novoAgente = { title: 'Neo', body: 'Desperto', userId: 1 };
3. Dispare o fetch assim:
const resposta = await fetch('https://jsonplaceholder.typicode.com/posts', {
method: 'POST',
body: JSON.stringify(novoAgente),
headers: { 'Content-type': 'application/json; charset=UTF-8' }
});
4. Converta a resposta para json (await resposta.json()) e exiba no cyber-output. Vai notar que a API devolve o objeto criado com um novo ID gerado no servidor!
