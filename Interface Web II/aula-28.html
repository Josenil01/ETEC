Aula 28: Os Caçadores de Relíquias

Desça às catacumbas do PostgreSQL. Aprenda o Padrão Repository (Data Access Layer) e isole o seu código SQL para proteger as suas Entidades de Domínio.

1. A Ilusão da Responsabilidade

Na Aula 20, escrevemos comandos SQL (INSERT, SELECT) diretamente no ficheiro principal do servidor (servidor.js). Na Aula 27, criámos Classes de Domínio fantásticas e isoladas. Mas levanta-se uma questão: Quem guarda a Entidade na Base de Dados?

A classe Reliquia não deve saber escrever SQL (isso quebra o encapsulamento). O ficheiro servidor.js (o Gestor de Rotas HTTP) também não deve sujar as mãos com tabelas, o seu trabalho é apenas receber a internet.

Para resolver este caos, a Arquitetura de Software moderna criou o Padrão Repository (Repositório). Pense nele como o nosso "Indiana Jones" ou "Lara Croft": o único especialista autorizado a descer às catacumbas do PostgreSQL, procurar a relíquia, limpá-la e entregá-la já formatada ao Museu (Servidor).

2. Mapeando as Ruínas

A Relíquia (O Domínio)

Apenas um Molde (Classe) puro. Contém as propriedades (nome, época, valor) e as regras do artefacto. Não tem NENHUMA ligação com base de dados.

class Reliquia { ... }
O Arqueólogo (Repository) ⭐

O Data Access Layer (DAL). Uma classe que importa a conexão do banco de dados (pool) e faz os comandos SQL SELECT e INSERT.

class ReliquiaRepository {
buscarPorId(id)
salvar(obj)
}
O Museu (Controller)

Recebe os pedidos da internet (Front-End). Ele chama o Repositório e pede: "Ei Jones, traz-me a relíquia nº 5". E depois devolve-a ao cliente em JSON.

app.get('/api/reliquias')

3. O Museu de Antiguidades (Preview)

No cenário abaixo, ao escavar um artefacto, o Front-End chama o Servidor. O Servidor não usa SQL! Ele delega o trabalho pesado para o ReliquiaRepository, que desce à Base de Dados e devolve a entidade guardada.

O Mapa das Ruínas (Passo Zero)

Antes de iniciar a escavação com o Node.js, é obrigatório criar o terreno. Vá ao SQL Editor do seu painel no Neon.tech e execute o seguinte comando para construir a tabela base:

CREATE TABLE reliquias (
  id SERIAL PRIMARY KEY,
  nome VARCHAR(100) NOT NULL,
  epoca VARCHAR(100) NOT NULL
);

/database/conexao.js 1. O Mapa para as Catacumbas (Ligação à BD)

Para que o Repositório consiga falar com o PostgreSQL, precisamos de configurar a ligação usando a chave secreta que guardamos no .env. Exportamos o pool para ser usado em qualquer parte do projeto.

conexao.js
const { Pool } = require('pg');
require('dotenv').config(); // Garante que o Node.js lê o ficheiro .env

// Cria a "poça" de conexões usando a nossa chave secreta do Neon.tech
const pool = new Pool({
    connectionString: process.env.DATABASE_URL,
    ssl: { rejectUnauthorized: false } // Exigência do Neon.tech para conexões seguras
});

// Exporta o pool para ser importado pelo Repositório
module.exports = pool;

/domain/Reliquia.js 2. O Molde do Artefacto

Tudo começa com a definição clara do que é uma Relíquia. Uma classe simples, sem dependências, que representa o nosso domínio de negócio e consolida o que aprendemos em POO.

Reliquia.js
class Reliquia {
    constructor(nome, epoca) {
        this.id = null; // Será preenchido pela base de dados
        this.nome = nome;
        this.epoca = epoca;
    }

    // Domínio Rico: O próprio objeto garante a sua validade!
    validar() {
        if (!this.nome || !this.epoca) {
            throw new Error("Dados corrompidos: Nome e Época são obrigatórios!");
        }
    }
}

// Exporta o molde para ser usado pelo Servidor e pelo Repositório
module.exports = Reliquia;

/repositories/ReliquiaRepository.js 3. O Isolamento do SQL

Este é o ÚNICO ficheiro da nossa aplicação que tem permissão para usar comandos SELECT ou INSERT. Ele isola a complexidade da base de dados e oferece métodos simples e semânticos (ex: salvar) ao restante da equipa.

Nota de Segurança: Repare que o uso de parâmetros estruturados como $1 e $2 nas queries serve como um escudo vital para prevenir ataques de SQL Injection (evitando que saqueadores de túmulos injetem código malicioso nas nossas ruínas).

ReliquiaRepository.js
// Importamos a ligação à BD e a classe do Domínio
const pool = require('../database/conexao.js');
const Reliquia = require('../domain/Reliquia.js');

class ReliquiaRepository {
    
    // 1. LISTAR TODOS (Read)
    async listarTodas() {
        // Adicionamos LIMIT 50 para evitar crash do servidor (Out of Memory) caso existam milhões de relíquias!
        const resultado = await pool.query('SELECT * FROM reliquias ORDER BY id DESC LIMIT 50');
        return resultado.rows; // Devolvemos apenas os dados limpos
    }

    // 2. SALVAR (Create)
    // Repare que ele recebe um objeto Reliquia pronto como parâmetro!
    async salvar(reliquiaObj) {
        const sql = 'INSERT INTO reliquias (nome, epoca) VALUES ($1, $2) RETURNING id';
        // O PostgreSQL substitui o $1 por nome, e o $2 por epoca
        const valores = [reliquiaObj.nome, reliquiaObj.epoca];
        
        const resultado = await pool.query(sql, valores);
        
        // Atualiza a entidade em memória com o ID gerado pelo Postgres
        reliquiaObj.id = resultado.rows[0].id;
        return reliquiaObj;
    }
}

// Exportamos a classe para o Servidor poder usar
module.exports = ReliquiaRepository;

servidor.js 4. O Express Elegante

Olhe para a beleza de um servidor sem comandos SQL espalhados! O código fica legível, limpo e parece que estamos a ler uma história em português.

servidor.js
const express = require('express');
const Reliquia = require('./domain/Reliquia.js');
const ReliquiaRepository = require('./repositories/ReliquiaRepository.js');

const app = express();
app.use(express.json());

// Instanciamos o nosso "Arqueólogo"
const repo = new ReliquiaRepository();

// Rota para listar tudo (O Museu pede as obras)
app.get('/api/reliquias', async (req, res) => {
    try {
        const catalogo = await repo.listarTodas();
        res.json(catalogo);
    } catch (erro) {
        console.error(erro);
        res.status(500).json({ erro: 'Falha interna no servidor.' });
    }
});

// Rota para Guardar Nova Relíquia
app.post('/api/reliquias', async (req, res) => {
    try {
        const { nome, epoca } = req.body;

        // 1. Domínio: Criamos o Molde (Objeto em Memória)
        const novoArtefato = new Reliquia(nome, epoca);

        // Validação Rica: A própria Entidade levanta um Erro se estiver inválida!
        novoArtefato.validar();

        // 2. Repositório: Pedimos ao Arqueólogo para enterrar no Banco de Dados
        const artefatoGuardado = await repo.salvar(novoArtefato);

        // 3. Retornamos o objeto com sucesso
        res.status(201).json(artefatoGuardado);
    } catch (erro) {
        console.error(erro);
        // Captura e devolve a mensagem do "throw new Error" ou um erro genérico
        res.status(400).json({ erro: erro.message || 'Falha interna no servidor.' });
    }
});

Front-End 3. A Expedição Dinâmica (cliente.js)

Para fechar o ciclo perfeito da Arquitetura Cliente-Servidor, criamos uma função carregarPrateleira() que faz o GET inicial assim que a página abre. Depois, no POST (ao submeter o formulário), otimizamos a rede usando `insertAdjacentHTML` para renderizar apenas a nova relíquia injetada, sem precisar recarregar todas as obras do museu!

cliente.js
const form = document.getElementById('dig-form');
const shelf = document.getElementById('museum-shelf');

// Centraliza a URL (Princípio DRY - Don't Repeat Yourself)
const API_URL = 'http://localhost:3000/api/reliquias';

// --- 1. O GET (Ler do Servidor e Desenhar no DOM) ---
async function carregarPrateleira() {
    try {
        const resposta = await fetch(API_URL);
        const reliquias = await resposta.json();
        
        shelf.innerHTML = ''; // Limpa a prateleira antes de desenhar
        
        if (reliquias.length === 0) {
            shelf.innerHTML = '<p class="col-span-2 text-center text-amber-700 italic">O museu está vazio. Inicie uma escavação!</p>';
            return;
        }

        reliquias.forEach(rel => {
            shelf.innerHTML += `
                <div class="relic-card">
                    <span class="relic-id">#${rel.id}</span>
                    <div class="relic-title">${rel.nome}</div>
                    <div class="text-xs text-amber-700 uppercase tracking-widest">${rel.epoca}</div>
                </div>
            `;
        });
    } catch (erro) {
        console.error('Erro ao carregar a prateleira:', erro);
    }
}

// --- 2. O POST (Gravar no Servidor via Formulário) ---
form.addEventListener('submit', async (e) => {
    e.preventDefault(); // Bloqueia o recarregamento da página

    const btnSubmit = form.querySelector('button');
    const textoOriginal = btnSubmit.innerText;

    // Bloqueia duplo clique e dá feedback visual
    btnSubmit.disabled = true;
    btnSubmit.innerText = "Escavando...";

    // Extrai os dados
    const artefacto = {
        nome: document.getElementById('input-artefacto').value,
        epoca: document.getElementById('input-epoca').value
    };

    try {
        // Dispara a expedição (POST) usando a constante
        const resposta = await fetch(API_URL, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(artefacto)
        });

        if (resposta.ok) {
            // Extrai o objeto devolvido pelo servidor (Graças ao RETURNING *)
            const novaReliquia = await resposta.json();
            
            // Injeta APENAS o novo card no início da prateleira, poupando a rede!
            const htmlNovoCard = `
                <div class="relic-card">
                    <span class="relic-id">#${novaReliquia.id}</span>
                    <div class="relic-title">${novaReliquia.nome}</div>
                    <div class="text-xs text-amber-700 uppercase tracking-widest">${novaReliquia.epoca}</div>
                </div>
            `;
            
            // Se houver a mensagem de "A escavar...", limpa-a primeiro
            if (shelf.querySelector('p')) shelf.innerHTML = '';
            
            // insertAdjacentHTML adiciona sem reescrever todo o conteúdo (muito mais rápido que .innerHTML +=)
            shelf.insertAdjacentHTML('afterbegin', htmlNovoCard);
            
            form.reset(); // Limpa os campos do formulário
        }
    } catch (erro) {
        console.error('Erro ao escavar:', erro);
    } finally {
        // Devolve o botão ao estado original e reativa-o
        btnSubmit.disabled = false;
        btnSubmit.innerText = textoOriginal;
    }
});

// 3. Inicia a expedição (GET) assim que a página abrir no navegador
carregarPrateleira();

4. Expedições Práticas

Nível: Estudante de Arqueologia (Fácil)

01

Leitura Específica: No seu ficheiro ReliquiaRepository.js, crie um novo método chamado buscarPorId(idAlvo). Lá dentro, escreva a lógica usando o pool.query com a query SELECT * FROM reliquias WHERE id = $1, retorne o rows[0].

Nível: Curador de Museu (Médio)

02

Proteção de Artefactos Vazios: No método buscarPorId que criou no exercício anterior, o que acontece se o utilizador passar um ID que não existe? Modifique o código do Repositório: use if (resultado.rowCount === 0) para que a função retorne null em vez de crashar a aplicação.

Nível: Indiana Jones (Desafio)

03

A Operação de Destruição (Delete):

1. No ReliquiaRepository.js, crie o método deletar(id) com o comando SQL DELETE FROM reliquias WHERE id = $1.
2. No servidor.js, crie uma rota app.delete('/api/reliquias/:id').
3. A rota do servidor deve capturar o req.params.id e simplesmente chamar o seu Arqueólogo: await repo.deletar(idRecebido), respondendo de seguida com status 200!

Dica de Integração: Para testar esta rota a partir do Front-End, o aluno usaria algo como fetch('http://localhost:3000/api/reliquias/1', { method: 'DELETE' }) no cliente.js.

Dica Avançada (O Assassino Silencioso): O comando DELETE não gera erro no banco de dados se tentar apagar um ID inválido (ele apenas afeta 0 linhas). Verifique se resultado.rowCount === 0 no Repositório para devolver um Erro 404 (Not Found)!
