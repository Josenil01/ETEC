Aula 25: O Treinamento Jedi

Viaje para Dagobah. Domine a POO Avançada com Atributos Privados, Parâmetros em Métodos, e a distinção mística entre Procedimentos e Funções.

1. A Força é Profunda

Na Aula 24, em Kamino, criámos clones básicos. Mas um Jedi não é um clone padrão. Um Jedi possui uma ligação profunda e íntima com a Força.

Falámos de Encapsulamento (não mudar propriedades de fora). Mas até agora, isso era apenas uma "promessa" entre programadores. Se alguém quisesse forçar luke.forca = 9999 no código, o JavaScript permitia. Hoje, o Mestre Yoda vai ensinar-nos a proteger as nossas Classes com os verdadeiros escudos da POO: os Atributos Privados. Além disso, veremos como os Métodos (as ações) podem receber parâmetros do mundo exterior para criar interações complexas entre Objetos.

2. Os Ensinamentos de Yoda

O Escudo Invisível (Atributos Privados)

Para garantir que ninguém manipula o nível de Força de um Jedi fora das regras de treino, trancamos a propriedade usando um cardinal (#) antes do seu nome.

class Padawan {
  #nivelForca; // 1. Declaração do campo privado!

  constructor(nome) {
    this.nome = nome; // Público (Qualquer um pode ler)
    this.#nivelForca = 10; // Privado!
  }
}

const luke = new Padawan('Luke');
console.log(luke.nome); // Funciona: "Luke"
luke.#nivelForca = 999; // ERRO FATAL: Private field!
Procedimento (Fazer)

Um Procedimento é um método que apenas executa uma ação ou altera o estado interno do objeto. Ele não devolve nenhum valor (não tem return).

// Recebe parâmetro, mas não tem return
treinar(horas) {
  this.#nivelForca += (horas * 5);
}
Função (Perguntar/Devolver)

Uma Função é um método que avalia dados e devolve um resultado (usa return). É assim que permitimos que o exterior "leia" os dados privados sem os conseguir modificar diretamente (Getters).

// Usado para perguntar o nível atual
consultarForca() {
  return this.#nivelForca;
}
A Ligação da Força (Interação entre Objetos)

Um Jedi nunca está sozinho. Na POO avançada, os métodos de um objeto podem receber outros objetos inteiros como parâmetros. Isso permite que instâncias interajam entre si, como num duelo de sabres de luz!

// O método ataca um Objeto (e não apenas um número ou string)
atacar(alvoInimigo) {
  // Podemos ler ou alterar as propriedades do objeto 'alvoInimigo'!
  alvoInimigo.vida -= 20;
  console.log(`Atingiste o Sith! A vida dele agora é ${alvoInimigo.vida}`);
}

POO Avançada 3. A Classe Jedi Protegida

Observe cuidadosamente como a classe abaixo declara #nivelForca no topo. Adicionámos também a validação defensiva no método treinarHabilidade para impedir que alguém envie "muitos" pontos (texto) ou números negativos e quebre o nosso estado interno.

Jedi.js
class Padawan {
    // 1. Declaração do Atributo Privado (Inacessível de fora)
    #nivelForca; 

    constructor(nome) {
        this.nome = nome;
        this.#nivelForca = 10; // Começa fraco
    }

    // FUNÇÃO (Getter): Devolve o valor privado em segurança
    consultarForca() {
        return this.#nivelForca;
    }

    // PROCEDIMENTO: Recebe um Parâmetro e muda o estado interno
    treinarHabilidade(pontosDeForca) {
        // VALIDAÇÃO DEFENSIVA: Protege a integridade do dado!
        if (typeof pontosDeForca !== 'number' || pontosDeForca <= 0) {
            console.error("Tentativa do Lado Negro: O valor de treino deve ser um número positivo!");
            return; // Bloqueia a execução se os dados forem lixo
        }

        this.#nivelForca += pontosDeForca;
        
        // Limita a Força a 100% máximo
        if (this.#nivelForca > 100) {
            this.#nivelForca = 100;
        }
    }

    // FUNÇÃO DE AVALIAÇÃO: Devolve true ou false
    tentarLevantarNave() {
        if (this.#nivelForca >= 80) {
            return true; // "Faz ou não faz. Tentativa não há."
        } else {
            return false; // Falhou
        }
    }
}

DOM & Eventos 4. A Ligação entre Objeto e Interface

No ficheiro de cliente, nós criamos a Instância do Padawan. Quando os botões do HTML são clicados, chamamos o método treinarHabilidade(X) passando argumentos (parâmetros) diferentes para cada botão!

app.js
// 1. Instanciar o Objeto Físico
const luke = new Padawan("Luke Skywalker");

// Função Utilitária para Sincronizar o Objeto JS com o HTML (DOM)
function atualizarBarraDeForca() {
    // Usa a Função de Consulta para ler o valor privado de forma segura
    const poderAtual = luke.consultarForca(); 
    
    // Altera o CSS Width para fazer a barra crescer
    document.getElementById('force-bar').style.width = `${poderAtual}%`;
    document.getElementById('force-display').innerText = `Força: ${poderAtual}%`;
}

// 2. Os Eventos que ativam os Procedimentos passando Parâmetros!
document.getElementById('btn-pedra').addEventListener('click', () => {
    luke.treinarHabilidade(10); // Passa o Argumento 10
    atualizarBarraDeForca();
});

document.getElementById('btn-yoda').addEventListener('click', () => {
    luke.treinarHabilidade(30); // Treino intenso: Passa 30
    atualizarBarraDeForca();
});

O Clímax 5. Executando a Avaliação (Nave)

Quando tentamos levantar a nave, o JS pergunta à classe "És capaz?". A classe devolve true ou false. O Front-End decide a animação com base nessa resposta rigorosa.

app.js (Parte 2)
const xWing = document.getElementById('x-wing-element');
const logger = document.getElementById('jedi-log');

document.getElementById('btn-nave').addEventListener('click', () => {
    const conseguiu = luke.tentarLevantarNave(); // Devolve true/false

    logger.classList.remove('hidden');

    if (conseguiu === true) {
        logger.innerHTML = "> A Força está contigo! A nave ergue-se do pântano!";
        logger.className = "mt-4 p-2 bg-black text-emerald-400 font-mono text-xs border border-emerald-700";
        
        // Manipulação de CSS Dinâmica (Adiciona a classe que levanta a nave)
        xWing.classList.add('levitating');
        document.getElementById('dagobah-bg').classList.add('force-active');
    } else {
        logger.innerHTML = "> Falhaste. O tamanho não importa, mas a Força atual não é suficiente. Treina mais.";
        logger.className = "mt-4 p-2 bg-black text-amber-500 font-mono text-xs border border-amber-700";
        
        // Efeito de abanão de falha (vibração ligeira opcional)
        xWing.style.transform = "translateX(5px)";
        setTimeout(() => xWing.style.transform = "translateX(0)", 200);
    }
});
