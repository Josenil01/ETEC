Aula 19: O Cofre de Gringotes

Abandone as bolsas mágicas temporárias. Descubra os Bancos de Dados na Nuvem, o PostgreSQL e como conectar o seu servidor Node.js ao cofre inquebrável do Neon.tech.

1. A Ilusão da Mochila (LocalStorage)

Até agora, utilizámos o localStorage e o sessionStorage para guardar as nossas informações. Eles são como o feitiço de Extensão Indetetável na bolsa da Hermione: são muito úteis, mas têm um grande problema.

O localStorage guarda os dados apenas no navegador do utilizador. Se o Harry gravar o "Mapa do Maroto" no telemóvel dele usando o `localStorage`, o Ron não conseguirá ver o mapa no computador dele! Para que os dados sejam partilhados por todos, em tempo real, e guardados com segurança máxima, precisamos de um cofre centralizado debaixo da terra: um Banco de Dados (Database).

2. Os Duendes e o Cofre

O Livro-Razão (PostgreSQL)

A tecnologia de Banco de Dados Relacional mais robusta e avançada do mundo. Pense nele como uma folha de Excel gigantesca e supersegura, guardada pelos duendes, onde a informação é organizada estritamente em Tabelas (linhas e colunas).

A Língua dos Duendes (SQL)

O PostgreSQL não entende JavaScript! Para falar com os duendes e pedir-lhes para guardar ou buscar dados, usamos a linguagem SQL (Structured Query Language). Exemplo: SELECT * FROM bruxos;.

A Filial Moderna de Gringotes (Neon.tech)

Ter um PostgreSQL instalado no nosso próprio computador não resolve o problema da partilha na internet. Precisamos de o colocar na Nuvem.

O Neon.tech é um serviço moderno que nos oferece um PostgreSQL na nuvem (Serverless) de forma rápida, gratuita e escalável. O nosso Servidor Node.js (Aula 14) irá ligar-se ao Neon para guardar os dados!

3. Abrindo o Seu Cofre (Setup)

Siga estas instruções como se fossem o mapa para a Pedra Filosofal. Hoje vamos garantir a chave do cofre e aprender como a ler!

Passo 1: A Inscrição no Banco

Aceda a neon.tech e crie uma conta gratuita. Pode usar a sua conta do GitHub ou Google.

Passo 2: Construir o Cofre

Após entrar, clique em "New Project". Dê-lhe um nome mágico (ex: gringotes-db), escolha a versão 16 ou superior do Postgres e clique em Create.

Passo 3: A Chave do Cofre (Connection String)

O Neon vai exibir-lhe uma linha de código vital. Essa é a sua Connection String. Não tenha medo dela, basta dissecá-la:

postgresql:// ron_weasley: SenhaSuperSecreta123@ ep-magic-wand.neon.tech/ gringotes
  • Protocolo: Avisa o sistema que estamos a falar Postgres.
  • Utilizador: O nome do duende autorizado.
  • Password: A senha de acesso (o segredo!).
  • Host (Servidor): A morada na internet onde o seu cofre está localizado.
  • Database: O nome do cofre específico dentro daquele servidor.

⚠ AVISO EXTREMO: Quem tiver esta String tem acesso a 100% da sua base de dados! Nunca a coloque em ficheiros Front-End (HTML ou cliente.js).

JavaScript 3. Lógica do Formulário (cliente.js)

cliente.js
const form = document.getElementById('vault-form');
const inputString = document.getElementById('input-string');
const vaultDoor = document.getElementById('vault-door');
const vaultMsg = document.getElementById('vault-msg');

form.addEventListener('submit', (e) => {
    e.preventDefault();
    const attempt = inputString.value.trim();

    // Reset das classes de mensagem
    vaultMsg.classList.remove('hidden', 'text-error', 'text-success');

    if(attempt === "") {
        vaultMsg.textContent = "⚠ O campo está vazio. Tente novamente.";
        vaultMsg.classList.add('text-error');
        vaultDoor.classList.remove('open');
        return;
    }

    // Simulação simples de validação de URL Postgres no Front-End
    if(attempt.startsWith('postgresql://')) {
        // Sucesso! Adicionamos a classe .open à porta
        vaultDoor.classList.add('open');
        vaultMsg.innerHTML = "Autenticação aceite! O cofre está aberto.";
        vaultMsg.classList.add('text-success');
    } else {
        // Falha! Removemos a classe .open
        vaultDoor.classList.remove('open');
        vaultMsg.textContent = "⚠ A Connection String deve começar por 'postgresql://'.";
        vaultMsg.classList.add('text-error');
    }
});

A Engenharia de Ligação (Back-End)

Servidor Local 4. Criar o ficheiro .env

Na pasta do seu projeto Node.js (onde está o seu servidor), crie um ficheiro novo chamado exatamente .env (sem nome antes do ponto). Cole lá dentro a string que o Neon lhe deu.

.env
# Este ficheiro nunca deve ser enviado para o GitHub!
DATABASE_URL="postgresql://seu_utilizador:sua_senha_secreta@ep-magic-wand-12345.eu-central-1.aws.neon.tech/gringotes-db?sslmode=require"

Terminal 5. O Leitor Mágico e o Carrinho dos Duendes

Precisamos instalar dois pacotes cruciais: o dotenv (para ler a chave em segredo) e o pg (node-postgres, que é o "carrinho" que viaja do nosso código até à base de dados na nuvem). No terminal, corra:

npm install dotenv pg

Back-End (Node) 6. Abrindo o Cofre no servidor.js

No código do nosso servidor, usamos o pacote pg para criar um "Pool" (uma poça de conexões prontas a usar) alimentado pela nossa chave secreta do .env. Para testar, faremos uma consulta simples: pedir a hora ao servidor da base de dados!

servidor.js
// 1. Invoca a magia que lê o ficheiro .env em segredo
require('dotenv').config();

const express = require('express');
const cors = require('cors'); 
// 2. Importa o "Carrinho" que fala com a Base de Dados
const { Pool } = require('pg'); 

const app = express();
app.use(cors());
app.use(express.json());

// 3. Configura a ligação usando a chave do .env
const pool = new Pool({
    connectionString: process.env.DATABASE_URL,
});

// 4. Rota para testar a conexão (O Feitiço Alohomora)
app.get('/api/teste-cofre', async (req, res) => {
    try {
        // Consulta a Base de Dados para saber a hora atual do servidor lá na nuvem!
        const resultado = await pool.query('SELECT NOW()');
        
        res.json({
            sucesso: true,
            mensagem: 'O cofre foi aberto com sucesso!',
            hora_do_banco: resultado.rows[0].now
        });
    } catch (erro) {
        console.error('As defesas do cofre bloquearam o acesso:', erro);
        res.status(500).json({ erro: 'Falha ao abrir o cofre.' });
    }
});

app.listen(3000, () => console.log('Servidor Gringotes online!'));

03

A Abertura Oficial:

No seu projeto real Node.js:
1. Instale as bibliotecas (npm install express cors dotenv pg).
2. Esconda a chave no seu ficheiro .env.
3. Adicione o código de Back-End no seu servidor.js e execute-o.
4. Abra o seu Navegador e digite http://localhost:3000/api/teste-cofre. Se o ecrã mostrar a mensagem de sucesso e a hora atual devolvida pela Nuvem da Neon.tech, parabéns, o cofre foi violado com sucesso!
