Aula 23: O Incinerador

A arte da destruição. Aprenda a operação de Eliminação (Delete) usando o método HTTP correto e manipulando o DOM para remover elementos visualmente.

1. A Eliminação de Dados (CRUD)

Chegámos à última peça do nosso puzzle. O seu inventário está cheio de blocos de terra inúteis e precisa de deitá-los na lava para libertar espaço. Eis a nossa sigla completa:

C Create (Criar)
R Read (Ler)
U Update (Atualizar)
D Delete (Apagar)

Hoje o foco é o Delete (Apagar). No Front-End usaremos o método HTTP DELETE. No Back-End, pediremos aos duendes do PostgreSQL para atirarem a linha da base de dados ao incinerador. Finalmente, veremos o método .remove() para apagar o HTML da vista do utilizador.

2. O Poço de Lava (Alerta Vermelho)

O Comando Destrutivo Supremo

Se achou que esquecer o WHERE num UPDATE era mau, fazê-lo num DELETE é o apocalipse digital. O comando DELETE FROM apaga linhas inteiras de uma só vez.

-- ⚠️ DESTRUIÇÃO TOTAL: Apaga absolutamente TUDO da tabela inventario!
DELETE FROM inventario;

-- ✅ CIRÚRGICO: Apaga apenas o bloco que tem a etiqueta (ID) número 15
DELETE FROM inventario WHERE id = 15;

Passo a Passo da Implementação

HTML 1. A Estrutura com IDs Dinâmicos

Para apagar algo, o Front-End precisa de saber qual elemento vai desaparecer. Note que a <div> principal tem o id="bloco-15".

Lembrando a "Aula 10 (Eventos)", não usamos o atributo inline onclick pois é uma prática antiga (legacy). O padrão moderno é criar uma classe genérica (ex: btn-destruir) e guardar o ID na propriedade data-id, para que o nosso JavaScript anexe os ouvintes de eventos (addEventListener) dinamicamente!

index.html
<!-- O ID do HTML combina um prefixo com o ID do banco de dados (ex: 15) -->
<div id="bloco-15" class="mc-inventory-row">
    <div class="mc-item-info">
        <span>💎</span>
        <p>Bloco de Diamante</p>
    </div>
    <!-- Usamos data-id para guardar o ID de forma invisível mas acessível ao JS! -->
    <button class="btn-lava btn-destruir" data-id="15">Destruir</button>
</div>

CSS 2. A Animação de Destruição

No seu style.css, preparamos a classe .burn-anim. Ela fará o item brilhar a laranja (fogo) e depois "encolher" até desaparecer (escondendo-o com `display: none` no final do Keyframe).

style.css
/* Classe ativada pelo JS após sucesso no servidor */
.burn-anim {
    animation: incinerate 0.8s forwards;
}

@keyframes incinerate {
    0% { transform: scale(1); }
    40% { 
        transform: scale(1.05); 
        background-color: #ea580c; /* Laranja Lava */
        filter: brightness(1.5) drop-shadow(0 0 10px #ea580c); 
    }
    80% { transform: scale(0.5) translateY(20px); opacity: 0.5; }
    100% { 
        transform: scale(0); 
        opacity: 0; 
        display: none; /* Some completamente do layout */
    }
}

Back-End (Node) 3. A Rota DELETE (servidor.js)

A Regra de Ouro (Padrão RESTful): URLs não devem conter verbos! Em vez de /api/destruir/:id, o correto é usar o substantivo /api/inventario/:id. O simples fato de usarmos app.delete() já indica a ação.

Além disso, adicionamos a validação de `NaN` (Not a Number). Se um Feiticeiro das Trevas enviar texto em vez de um número (ex: /api/inventario/avada), a conversão `parseInt` vai falhar. O nosso if(isNaN()) impede um erro 500 catastrófico no banco!

servidor.js
// Rota DELETE seguindo o padrão RESTful (URL é Substantivo, Método é Verbo)
app.delete('/api/inventario/:id', async (req, res) => {
    try {
        // 1. Capturamos e convertemos o ID alvo para Número Inteiro
        const idAlvo = parseInt(req.params.id);

        // VALIDAÇÃO DE DEFESA: Se enviarem texto ('avada-kedavra'), o parseInt devolve NaN
        if (isNaN(idAlvo)) {
            return res.status(400).json({ erro: 'O ID fornecido não é válido.' });
        }

        // 2. O Comando de Destruição (Com o WHERE protetor!)
        const sql = 'DELETE FROM inventario WHERE id = $1';
        
        // 3. Executamos na base de dados
        const resultado = await pool.query(sql, [idAlvo]);

        // VERIFICAÇÃO CRÍTICA: Se apagou 0 linhas, é porque o ID não existia
        if (resultado.rowCount === 0) {
            return res.status(404).json({ erro: 'Item não encontrado. Já foi apagado?' });
        }

        // Tudo certo! (Código 200 OK é padrão)
        res.json({ msg: `Bloco com ID ${idAlvo} destruído para sempre.` });

    } catch (erro) {
        console.error(erro);
        res.status(500).json({ erro: 'Falha técnica no incinerador.' });
    }
});

JS Cliente 4. O Disparo e a Magia do DOM (remove vs hidden)

Usamos querySelectorAll para ouvir os cliques em todos os botões e pegamos no data-id. Se o servidor confirmar a destruição, usamos o método .remove().

Porquê .remove()? Muitos iniciantes dariam apenas uma classe .hidden (CSS display: none). Porém, esconder elementos no CSS mantém-nos vivos na memória do navegador (Mundo Invertido). Se esconder 1000 blocos, o PC vai ficar lento (Memory Leak). O .remove() destrói o elemento fisicamente, libertando memória RAM!

cliente.js
// 1. Selecionamos todos os botões da classe "btn-destruir"
const botoesLava = document.querySelectorAll('.btn-destruir');

// 2. Para cada botão, adicionamos o ouvinte de eventos moderno
botoesLava.forEach(btn => {
    btn.addEventListener('click', async (event) => {
        
        // 3. Lemos o ID guardado secretamente no HTML (data-id)
        const idRecebido = event.target.getAttribute('data-id');
        const blocoHTML = document.getElementById(`bloco-${idRecebido}`);

        try {
            // O método HTTP é DELETE, seguindo o padrão RESTful na URL
            const res = await fetch(`http://localhost:3000/api/inventario/${idRecebido}`, {
                method: 'DELETE'
            });

            const dados = await res.json();
            if (!res.ok) throw new Error(dados.erro);
            
            // --- SUCESSO! A base de dados já apagou. ---
            
            blocoHTML.classList.add('burn-anim');
            
            setTimeout(() => {
                // Evitando Memory Leaks: Apaga definitivamente em vez de apenas esconder!
                blocoHTML.remove(); 
            }, 800);

        } catch (erro) {
            alert(`O bloco resistiu: ${erro.message}`);
        }
    });
});

4. Desafios da Destruição

01
Cuidado com o Alvo (SQL WHERE):

Escreva num bloco de notas o comando SQL exato que os duendes devem executar para apagar apenas o registo onde a coluna material seja igual a "Terra". O que aconteceria se não incluísse a cláusula de restrição?

02
A Confirmação (UX):

Apagar dados é uma ação perigosa. No cliente.js (Passo 4), antes mesmo do bloco try, adicione a linha: if (!confirm("Tem certeza que deseja incinerar isto?")) return;. Esta função nativa do JS vai gerar um popup do navegador. Se o utilizador clicar "Cancelar", o código para (return) e o item é salvo!

03
Desafio Épico: O "Soft Delete"

Na vida real (Instagram, Bancos), os dados raramente são apagados de verdade. Em vez disso, usam uma técnica chamada Soft Delete (Falso Apagar).

1. Vá à consola da base de dados e adicione uma coluna na tabela: ALTER TABLE inventario ADD COLUMN apagado BOOLEAN DEFAULT false;
2. No seu servidor.js, altere o comando de DELETE FROM... para um comando de atualização que mude apenas esse estado: UPDATE inventario SET apagado = true WHERE id = $1.
3. A partir de agora, o dado ainda existe lá no banco! Você apenas o marcou como inativo. (Dica: Terá de atualizar a sua rota de Leitura (Aula 21) para fazer SELECT * FROM inventario WHERE apagado = false, assim as coisas "apagadas" não aparecem no ecrã).
