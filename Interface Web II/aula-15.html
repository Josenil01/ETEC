Aula 15: O Pombo Correio

Aprenda a enviar os dados dos seus formulários Front-End para o Servidor usando o protocolo HTTP POST, como se estivesse a enviar uma coruja para Hogwarts.

1. O Correio Coruja

Na Aula 11 nós criámos formulários e validámos os dados para não deixar os muggles (trouxas) passarem. Na Aula 14, descobrimos como criar um Servidor Node.js para receber pedidos. Agora, vamos ligar as duas pontas!

Quando um feiticeiro preenche um pergaminho (HTML Form), ele não pode simplesmente atirá-lo ao ar. É preciso usar um método de entrega (o `fetch` no JavaScript) e especificar a morada do destinatário (a URL do Servidor) e como a carta está selada (os `headers` e o `body` JSON).

2. A Arte de Selar a Carta

1. Parar o Correio Muggle
event.preventDefault()

Lembre-se da regra de ouro! Se não impedir o envio padrão do formulário HTML, a página recarrega e o seu código JavaScript nunca chegará a enviar a coruja digital.

2. Selando o Envelope (JSON)
JSON.stringify(dados)

O servidor não entende objetos JavaScript puros. Precisamos transformar o nosso objeto (com os campos do formulário) numa string (texto puro) para viajar pela rede.

3. As Instruções da Coruja (Fetch Options)
method: 'POST' Avisa que estamos a enviar um pacote novo, e não apenas a pedir dados.
headers O "rótulo". Avisamos o servidor: 'Content-Type': 'application/json' para ele saber o que vem na carta.
body O conteúdo exato da carta (o resultado do nosso JSON.stringify).

3. Exemplo Prático: Execução Local

Escreva no pergaminho abaixo e clique em enviar. O Front-End irá capturar o formulário, criar um pacote JSON e disparar um fetch simulando o envio para o nosso Servidor Node.js.

Camada 1 (Front-End) O Despacho da Carta (cliente.js)

No ficheiro cliente.js, capturamos o evento submit, extraímos o .value dos campos e empacotamos tudo na propriedade body do fetch.

cliente.js
const form = document.getElementById('hogwarts-form');
const output = document.getElementById('server-output');

form.addEventListener('submit', async (event) => {
    event.preventDefault(); // 1. Bloqueia o envio muggle (refresh)

    // 2. Lê os valores escritos pelo utilizador
    const nomeFeiticeiro = document.getElementById('input-remetente').value;
    const mensagemEscrita = document.getElementById('input-mensagem').value;

    // Criamos o objeto que queremos enviar
    const carta = {
        remetente: nomeFeiticeiro,
        conteudo: mensagemEscrita
    };

    output.classList.remove('hidden');
    output.innerHTML = '<span class="flying-owl">✉️</span> Coruja a caminho...';

    // 3. A Coruja Levanta Voo (Fetch para o localhost na porta 3000)
    const resposta = await fetch('http://localhost:3000/api/mensagens', {
        method: 'POST',
        headers: { 
            'Content-Type': 'application/json' // Avisa que o formato é JSON
        },
        body: JSON.stringify(carta) // Transforma o objeto em texto selado
    });

    // 4. Aguarda e lê o que o Servidor respondeu
    const dadosServidor = await resposta.json();
    output.innerHTML = `<strong>Hogwarts diz:</strong> ${dadosServidor.resposta}`;
});

Camada 2 (Back-End) O Ficheiro servidor.js

No código do servidor Node.js (que corre no seu terminal), recebemos o req.body, processamos a carta e respondemos via res.json().

servidor.js (Ficheiro Node.js/Express)
const express = require('express');
const cors = require('cors'); 

const app = express();
app.use(cors()); 
app.use(express.json()); // Essencial para o Express ler o JSON que vem no Body!

// Rota para receber as corujas
app.post('/api/mensagens', (req, res) => {
    
    // O req.body vai conter exatamente o objeto 'carta' que enviámos no Front-End!
    const remetenteChegou = req.body.remetente;
    const mensagemChegou = req.body.conteudo;
    
    console.log(`Coruja recebida de: ${remetenteChegou}`);

    // Prepara a resposta oficial para enviar de volta ao Front-End
    let textoResposta = `Caro(a) ${remetenteChegou}, a sua carta foi entregue no Grande Salão com sucesso.`;

    if (remetenteChegou.toLowerCase().includes('voldemort')) {
        textoResposta = 'ALERTA: Acesso negado Àquele-Que-Não-Deve-Ser-Nomeado!';
    }

    // Envia a resposta final em formato JSON
    res.json({ resposta: textoResposta });
});

app.listen(3000, () => console.log('Hogwarts à escuta na porta 3000'));

4. Exames N.I.E.M. (Prática)

Nível: Aluno do 1º Ano (Fácil)

01

O Cabeçalho (Headers): No código do cliente.js (Passo 2), escrevemos 'Content-Type': 'application/json'. Tente remover apenas essa linha do objeto headers do seu código local e faça um envio. O que acontece no servidor? (Dica: o servidor não vai conseguir ler o req.body corretamente sem esse aviso!).

Nível: Monitor (Médio)

02

Adicionando a Casa: Adicione um elemento <select id="input-casa"> no seu HTML com opções (Gryffindor, Slytherin, etc). Modifique o cliente.js para ler também o valor desse select, adicioná-lo ao objeto carta e enviá-lo no POST. Verifique se no `console.log()` do servidor a casa do aluno aparece corretamente.

Nível: Auror (Desafio)

03

O Berrador (Tratamento de Erros): E se a magia falhar? No cliente.js, adicione um bloco try { ... } catch(erro) { ... } em volta de todo o processo de `fetch`.

Para simular uma falha (como o servidor estar desligado), pare o terminal do Node.js (Ctrl+C). Ao clicar no botão do Front-End, o catch deve disparar e imprimir no ecrã: "A coruja perdeu-se! Servidor offline." com letras vermelhas.
