1. A Poção Corrompida (O Perigo dos Dados Sujos)

Na Aula 31, o Batman ensinou-nos a usar as Consultas Parametrizadas. Elas são excelentes para evitar que o banco confunda dados com comandos (SQL Injection). Mas e se o invasor enviar dados que não são comandos, mas sim "lixo" ou dados fora do formato esperado?

Imagine os Comensais da Morte a enviar formulários para o Ministério da Magia. O sistema pede a idade do bruxo (um número), mas eles enviam o texto "AvadaKedavra". O banco tenta gravar um texto numa coluna numérica e explode a aplicação (Crash). Ou pior, enviam um nome cheio de espaços em branco e caracteres invisíveis.

// O servidor espera um número limpo para calcular a idade:
let anoNascimento = 2026 - req.body.idade;

// O que acontece quando o Comensal envia texto na idade:
let anoNascimento = 2026 - "Sonserina"; // Resultado: NaN (Not a Number) -> O sistema colapsa!

A Parametrização não resolve isto! Precisamos de uma primeira linha de defesa: a Validação (perguntar "Isto é realmente um número?") e a Sanitização (lançar um feitiço de limpeza para remover espaços em branco e sujidade).

3. O Livro de Feitiços (As Ferramentas de Defesa)

As linguagens de programação possuem feitiços nativos (funções) para tratar dados sujos. No Node.js (JavaScript), temos ferramentas essenciais para limpar e validar dados antes de sequer tentarmos inseri-los no Neon.tech.

🔍 Feitiço de Validação (Revelio)

Verifica se o dado é o que diz ser. É um número? É um email válido? Não altera o dado, apenas aprova ou bloqueia.

let idade = req.body.idade;

// Verifica se não é um número numérico válido
if (isNaN(idade)) {
    return res.send("A idade tem que ser número!");
}

🧹 Feitiço de Sanitização (Evanesco)

Limpa o dado. Remove espaços acidentais no início e no fim, retira tags HTML ou formata o texto corretamente.

let nome = req.body.nome;

// .trim() remove espaços em branco inúteis
// .replace() remove tags maliciosas
nome = nome.trim(); 
nome = nome.replace(/<[^>]*>?/g, '');

4. Laboratório Real: A Proteção do Cofre

Vamos construir um servidor Node.js que recebe o registo de novos bruxos. Vamos implementar duas rotas POST: uma que aceita os dados de forma irresponsável e outra que lança os feitiços de segurança.

1 Prepare o Cofre no Neon.tech

Abra o SQL Editor do seu projeto Neon e rode o comando abaixo para criar a tabela do Ministério:

CREATE TABLE bruxos_registados (
    id SERIAL PRIMARY KEY,
    nome_completo VARCHAR(100),
    idade INT
);

Agora, crie o server.js. Preste atenção à forma como a Rota Limpa atua como um escudo antes de realizar o INSERT:

server.js
const express = require('express');
const { Pool } = require('pg');
const app = express();

// Middleware para ler formulários POST
app.use(express.urlencoded({ extended: true }));

const pool = new Pool({ connectionString: 'postgres://usuario:senha@neon.tech/neondb' });

// 🔴 ROTA SUJA: Guarda lixo sem verificar nada!
app.post('/registo-sujo', async (req, res) => {
    const nome = req.body.nome;
    const idade = req.body.idade;
    
    // Mesmo usando parametrização (protegendo de SQL Injection), 
    // se a idade for a palavra "Sonserina", o banco dará erro ao tentar gravar no tipo INT!
    try {
        await pool.query(`INSERT INTO bruxos_registados (nome_completo, idade) VALUES ($1, $2)`, [nome, idade]);
        res.send("Registado (de forma perigosa)!");
    } catch(err) { res.send("ERRO FATAL (Crash): " + err.message); }
});

// 🟢 ROTA LIMPA: Escudo Protetor Ativado
app.post('/registo-limpo', async (req, res) => {
    let nome = req.body.nome;
    let idade = req.body.idade;
    
    // --- 1. VALIDAÇÃO (Revelio) ---
    if (!nome || nome.trim() === '') {
        return res.send("Erro: O nome é obrigatório!");
    }
    if (isNaN(idade) || idade < 0 || idade > 150) {
        return res.send("Erro: Idade inválida enviada pelos Comensais!");
    }

    // --- 2. SANITIZAÇÃO (Evanesco) ---
    nome = nome.trim(); // Corta espaços
    nome = nome.replace(/<[^>]*>?/g, ''); // Remove tentativas de inserir tags HTML/Script
    
    // Tudo seguro? Agora sim gravamos!
    try {
        await pool.query(`INSERT INTO bruxos_registados (nome_completo, idade) VALUES ($1, $2)`, [nome, idade]);
        res.send(`<h1 style="color:green">Sucesso! Bem-vindo, ` + nome + `</h1>`);
    } catch(err) { res.send("Erro no banco: " + err.message); }
});

app.listen(3000, () => console.log('Ministério da Magia online na porta 3000'));

🛡️ 4. Teste o Escudo Protetor

  • Idade: Maldicao

Resultado: A requisição é parada logo na Validação ("Idade inválida!"). Se puser uma idade certa, o sistema grava apenas "Lúcio", destruindo os espaços em branco e arrancando a tag script indesejada.
