Aula 10: A Magia dos Eventos

Aprenda a fazer sua página "escutar" feitiços. Descubra como cliques, movimentos de mouse e digitação podem disparar reações dinâmicas usando o JavaScript.

1. O Livro de Feitiços: Ação e Reação

Até agora, nossa página era como um quadro na parede: bonita, mas inerte. No mundo bruxo, para que a magia aconteça, você precisa de um movimento de varinha (Ação) para gerar um feitiço visual (Reação).

Na web, chamamos as ações do usuário de Eventos. Clicar em um botão, passar o mouse por cima de uma imagem, ou digitar em um campo de texto são eventos. O JavaScript possui "ouvidos" mágicos que ficam esperando essas ações acontecerem para, então, executar um bloco de código (a reação).

2. O Mapa do Maroto dos Eventos

A Varinha Mágica
elemento.addEventListener('evento', função)

É o método que "anexa" um ouvido ao elemento. O primeiro parâmetro é o que ele deve escutar (ex: 'click'). O segundo é a função (o feitiço) que será executada quando o evento ocorrer.

Eventos de Mouse

'click': Quando o elemento é clicado.

'mouseenter': Quando o cursor do mouse entra no elemento.

'mouseleave': Quando o cursor sai do elemento.

Eventos de Teclado

'keydown': Assim que a tecla é pressionada (antes de soltar).

'keyup': Quando o usuário solta a tecla.

Eventos de Formulário

'input': Dispara em tempo real, a cada letra digitada num campo.

'submit': Quando um formulário tenta enviar os dados.

Passo 4 O Evento 'click' (Feitiço Lumos)

Adicionamos o "ouvido" ao botão. Quando ele é clicado, a função muda a classe da sala inteira, acendendo a luz.

magia.js
// 1. Escutando o CLIQUE do mouse
btnLumos.addEventListener('click', () => {
    // toggle: se tem a classe ele tira, se não tem ele coloca!
    salaPrecisa.classList.toggle('lumos-active');
    
    // Muda o texto do botão de acordo com a luz
    if (salaPrecisa.classList.contains('lumos-active')) {
        btnLumos.innerText = 'NOX!';
    } else {
        btnLumos.innerText = 'LUMOS!';
    }
});

Passo 5 O Evento 'mouseenter' (Pomo de Ouro)

Sempre que o mouse tenta entrar na área do pomo, o evento dispara, sorteia novos valores para o CSS top e left, fazendo a bolinha fugir.

magia.js
// 2. Escutando a ENTRADA do mouse
pomoDeOuro.addEventListener('mouseenter', () => {
    // Math.random() gera um número entre 0 e 1
    const novaPosicaoX = Math.random() * 80; // %
    const novaPosicaoY = Math.random() * 80; // %

    // Altera a posição CSS do elemento
    pomoDeOuro.style.left = `${novaPosicaoX}%`;
    pomoDeOuro.style.top = `${novaPosicaoY}%`;
});

Passo 6 O Evento 'input' (A Resposta do Diário)

O evento input é disparado em tempo real a cada tecla digitada. Nós pegamos o valor do campo de texto e o espelhamos em um parágrafo abaixo dele, como se o livro estivesse respondendo.

magia.js
// 3. Escutando a DIGITAÇÃO no campo de texto
// Note que a função recebe o 'event' (o relatório do que aconteceu)
diarioInput.addEventListener('input', (event) => {
    
    // event.target é o elemento que sofreu a ação (o input)
    // .value pega o que está escrito dentro dele
    const textoDigitado = event.target.value;

    if (textoDigitado === "") {
        diarioResposta.innerText = '...';
    } else {
        // O diário responde de forma fantasmagórica!
        diarioResposta.innerText = `O diário absorveu: "${textoDigitado}"`;
    }
});
