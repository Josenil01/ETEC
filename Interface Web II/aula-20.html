Aula 20: O Criador de Mundos

Defina as regras do seu universo com CREATE TABLE, evite a Magia Negra (SQL Injection) e aprenda a operação de Spawn (Create) usando INSERT.

1. O Spawn de Dados (CRUD)

Programar é, essencialmente, gerir informação. Para isso, usamos uma sigla mundialmente famosa: CRUD.

C Create (Criar)
R Read (Ler)
U Update (Atualizar)
D Delete (Apagar)

Hoje vamos focar no Create. Na base de dados, isso significa pegar num formulário do Front-End e "spawnar" uma nova linha (row) numa tabela. Sem o Create, o seu mundo estaria sempre vazio.

2. Gerando o Terreno (CREATE TABLE)

O Esquema do Mundo e a Representação Visual

Antes de qualquer coisa, precisamos de executar este comando uma única vez no editor do Neon.tech (aba SQL Editor) para criar a nossa tabela com regras fixas:

CREATE TABLE inventario (
  id SERIAL PRIMARY KEY, -- ID automático e único
  material VARCHAR(50) NOT NULL,
  quantidade INTEGER DEFAULT 1
);

O tipo SERIAL é mágico: ele gera um número novo (1, 2, 3...) automaticamente. Por causa disso, nunca enviamos o ID no comando de criar (`INSERT`), o banco resolve isso por nós!

JS Cliente 3. Gatilho de Invocação (fetch)

No cliente.js, capturamos o envio, empacotamos o material e a quantidade num JSON e enviamos para o nosso servidor local.

cliente.js
const form = document.getElementById('mc-form');

form.addEventListener('submit', async (e) => {
    e.preventDefault();
    
    const novoItem = {
        material: document.getElementById('mc-material').value,
        qtd: document.getElementById('mc-quantidade').value
    };

    const res = await fetch('http://localhost:3000/api/spawn', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(novoItem)
    });

    const dados = await res.json();
    
    if(res.ok) {
        alert(`Bloco #${dados.id} spawnado!`);
    } else {
        alert(`Erro: ${dados.erro}`);
    }
});

Back-End (Node) 4. Segurança e Criação no servidor.js

Aqui reside a inteligência central. Repare nos 4 pilares adicionados a este código:

  • express.json(): Sem ele, o servidor não percebe o pacote JSON que o Front-End enviou e o `req.body` fica vazio.
  • Validação (If): Evita que o programa tente gravar blocos vazios, o que causaria um Erro 500 no banco de dados.
  • Proteção SQL ($1, $2): Previne a SQL Injection (Creeper Hackers). Nunca concatene a string diretamente (ex: VALUES ('${material}')), pois o usuário pode injetar código malicioso. Os $1 limpam tudo.
  • RETURNING id: Exige que o banco devolva a etiqueta única que gerou para o novo bloco.
servidor.js
// ... (Setup anterior: express, cors, pg pool) ...
// OBRIGATÓRIO: O tradutor JSON (Se esquecer isto, o req.body vem undefined!)
app.use(express.json());

app.post('/api/spawn', async (req, res) => {
    const { material, qtd } = req.body;

    // 1. ESCUDO DE VALIDAÇÃO (Previne Crash na Base de Dados)
    if (!material || !qtd) {
        return res.status(400).json({ erro: 'Campos em branco não são permitidos!' });
    }

    try {
        // 2. A QUERY PROTEGIDA (Prevenção de SQL Injection) e RETURNING id
        // Não passamos o ID aqui! O 'SERIAL' trata disso sozinho.
        const sql = 'INSERT INTO inventario (material, quantidade) VALUES ($1, $2) RETURNING id';
        
        // O PostgreSQL substitui o $1 por material, e o $2 pela qtd convertida em número (parseInt seguro)
        const resultado = await pool.query(sql, [material, parseInt(qtd)]);
        
        const novoId = resultado.rows[0].id;

        res.status(201).json({ 
            msg: 'Item spawnado!', 
            id: novoId 
        });
    } catch (erro) {
        console.error(erro);
        res.status(500).json({ erro: 'Falha no spawn de blocos.' });
    }
});

4. Desafios do Criativo

01

Data Types: Vá ao Neon.tech e crie uma tabela chamada jogadores com: um ID (serial), um Nome (varchar) e uma coluna online do tipo BOOLEAN. Como ficaria o comando SQL (CREATE TABLE)?

02

Escudo Quebrado: No servidor.js, comente (desative) temporariamente a linha de validação `if (!material || !qtd)`. Use ferramentas como o Postman ou o ThunderClient para enviar um POST sem os dados. Observe a sua consola do servidor "crashar" com um Erro 500 do Postgres devido à violação do NOT NULL. Volte a ligar a validação para proteger o sistema e receber um Erro 400 controlado!
