Aula 18: Crachá de Acesso

Junte-se à S.H.I.E.L.D. Aprenda a gerir "Sessões", a renderizar interfaces com base em Níveis de Acesso, e descubra a grande ilusão de segurança do Front-End.

1. A Ilusão da Segurança

Você não pode simplesmente entrar no Triskelion (Sede da S.H.I.E.L.D.) sem um crachá de identificação. Na web, o processo de "Login" cria o que chamamos de uma Sessão. O JavaScript verifica as suas credenciais no sessionStorage e redesenha a página na hora (esconde o formulário, mostra o painel). A isto chamamos Controlo de Estado (State Management).

Mas há um segredo sombrio que todo arquiteto de software deve saber: O Front-End (HTML/CSS/JS) não é seguro. Esconder uma secção da tela usando CSS (ex: class="hidden") serve apenas para a Experiência do Utilizador (UX). Um hacker pode facilmente abrir as ferramentas de desenvolvedor (F12) e forçar a exibição do painel. A verdadeira segurança sempre acontece no Back-End (Servidor), que se recusa a enviar os dados se o crachá for inválido!

2. A Engenharia de Autorização

O Crachá Complexo (JSON)

Na vida real, a sessão não guarda apenas um nome. Precisamos de saber o Nível de Acesso. Para isso, guardamos um Objeto JSON na sessão. Assim, o nosso JS pode ler agente.nivel e decidir quais os botões que devem aparecer na tela (Renderização Condicional).

Controlo de Estado (UI State)

Não precisamos de criar páginas HTML diferentes (um login.html e um painel.html). Num ambiente moderno, temos os dois blocos no mesmo HTML e usamos a classe .hidden ativada pelo JavaScript para esconder um e mostrar o outro com base na existência do crachá.

JS Emitindo o Crachá Complexo

No momento do login, capturamos o nome e o cargo. Transformamos isto num Objeto JavaScript e usamos o JSON.stringify() para gravar essa estrutura de forma segura no sessionStorage.

script.js (Parte 1)
const formAuth = document.getElementById('form-auth');
const inputAgente = document.getElementById('input-agente');
const inputCargo = document.getElementById('input-cargo');

// --- O PROCESSO DE LOGIN ---
formAuth.addEventListener('submit', (event) => {
    event.preventDefault(); 
    
    // Criamos o "Crachá" com as propriedades do utilizador
    const crachaAgente = {
        nome: inputAgente.value,
        nivel: parseInt(inputCargo.value)
    };

    // Guarda na Sessão convertendo o Objeto para String JSON
    sessionStorage.setItem('shield_auth', JSON.stringify(crachaAgente));
    
    inputAgente.value = '';
    verificarSessao(); // Atualiza a interface
});

JS Renderização Condicional da Interface

Na função que redesenha a tela, extraímos o JSON da memória e usamos lógica condicional (if) para decidir se o botão do Protocolo Insight (Nível 10) deve aparecer.

script.js (Parte 2)
// Função central que controla a UI baseada no Estado
function verificarSessao() {
    const dadosSessao = sessionStorage.getItem('shield_auth');

    if (dadosSessao) {
        // Converte o texto JSON de volta para Objeto JS
        const agente = JSON.parse(dadosSessao);

        // Esconde o login, mostra o painel.
        document.getElementById('tela-login').classList.add('hidden');
        document.getElementById('tela-painel').classList.remove('hidden');
        document.getElementById('texto-boas-vindas').innerText = `Bem-vindo, ${agente.nome}.`;

        // RENDERIZAÇÃO CONDICIONAL POR NÍVEL
        const areaDiretor = document.getElementById('area-diretor');
        if (agente.nivel === 10) {
            areaDiretor.classList.remove('hidden'); // Mostra os botões vermelhos
        } else {
            areaDiretor.classList.add('hidden'); // Bloqueia visualmente
        }

    } else {
        // Não tem sessão! Garante que o login aparece e o painel esconde.
        document.getElementById('tela-login').classList.remove('hidden');
        document.getElementById('tela-painel').classList.add('hidden');
    }
}

// Executa a verificação mal a página carrega!
document.addEventListener('DOMContentLoaded', verificarSessao);

4. Treino e Cibersegurança

Nível: Agente Base (Fácil)

01

Inspeção Visual: Faça login no simulador acima como Agente de Campo. Depois, prima F12, vá à aba Application > Session Storage. Repare como o seu nome e o nível `1` estão lá gravados em formato de texto JSON. Mude a aba do navegador ou faça Refresh; a sua sessão persiste!

Nível: Hacker Infiltrado (Médio)

02

A Ilusão do Front-End (Hackeando a Interface):

Faça login como "Agente de Campo" (O botão vermelho do Protocolo Insight não deve aparecer).

Agora, vamos invadir o sistema da S.H.I.E.L.D.:
1. Prima F12. Vá à aba Elements (Elementos).
2. Pressione Ctrl+F e procure por area-diretor.
3. Você encontrará <div id="area-diretor" class="hidden">.
4. Dê dois cliques na palavra hidden e elimine-a. Prima Enter.

Magia! O painel vermelho super-secreto apareceu no seu ecrã! Isto prova que o Front-End pode ser adulterado pelo utilizador. É por isso que a verdadeira segurança de dados tem de acontecer SEMPRE no Servidor (Back-End), recusando pedidos `fetch` caso a sessão não seja válida do lado do banco de dados.

03

Injeção na Sessão: Outra forma de hackear é ir à aba Application (F12). Edite o campo Value do seu JSON no SessionStorage e mude o "nivel":1 para "nivel":10 manualmente. Faça um refresh à página (F5) e veja a interface renderizar os botões do Diretor como se você tivesse permissão!

Nível: Tony Stark (Desafio)

04

Timeout de Segurança: Por motivos de segurança militar, a sessão deve expirar sozinha. No seu JS, após fazer o login e abrir a tela do painel, utilize a função setTimeout() para que, passado exatos 1 minuto (60000 milissegundos), o JavaScript ative automaticamente a função de Logout, limpando o sessionStorage e atirando o utilizador de volta para o formulário de entrada, mostrando o `alert("Sessão expirada por motivos de segurança.")`.
