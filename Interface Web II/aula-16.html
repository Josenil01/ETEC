Aula 16: Defesa Contra as Artes das Trevas

Nunca confie no cliente. Aprenda a aplicar a Validação Dupla no Servidor, compreenda os Códigos de Status HTTP e lide com falhas de rede usando try/catch.

1. A Ilusão da Segurança

Na Aula 11 (Vingadores), fizemos a validação de formulários no Front-End. Mas um Feiticeiro das Trevas pode facilmente usar ferramentas de desenvolvimento para ignorar o seu JavaScript e enviar uma carta vazia ou maliciosa diretamente para o seu Servidor.

A regra de ouro da Cibersegurança é: "Nunca confies no cliente". O Front-End valida para dar uma boa experiência ao utilizador honesto, mas o Back-End (Servidor) valida novamente para proteger o sistema. Além disso, precisamos de saber como reagir quando a magia falha (ex: a internet vai abaixo) e como o Servidor nos comunica o que correu mal através de Códigos de Status.

2. Os Escudos de Proteção

A Linguagem do Ministério (HTTP Status Codes)
200 OK Sucesso absoluto. O feitiço foi recebido e processado corretamente.
400 Bad Request Erro do Cliente. Faltaram dados ou o formato da mensagem está inválido.
403 Forbidden Acesso Negado. O servidor percebeu a mensagem, mas recusa-se a autorizá-la (ex: Magia Negra).
500 Server Error Erro do Servidor. O código do Back-End "explodiu" a tentar processar o pedido.
A Ferramenta: FormData

Ir buscar o .value de cada input é exaustivo. O FormData varre todo o formulário de uma vez e recolhe todos os campos que tenham o atributo name.

const formObj = new FormData(form); const dados = Object.fromEntries(formObj);
A Rede de Segurança: Try/Catch

O que acontece se a internet do utilizador for abaixo? O fetch falha miseravelmente. Envolvemos a magia num bloco try, e se falhar, o bloco catch é ativado para avisar o utilizador em vez de congelar o ecrã.

3. Exemplo Prático: Execução Local

Neste simulador, o Front-End não valida se os campos estão vazios (removam os atributos `required` propositadamente). É o Servidor que fará a Validação Dupla.

2. Nome: Voldemort (Gera 403).
3. Feitiço: Avada Kedavra (Gera 500).
4. Ligar a caixa "Simular Falha de Rede" (Ativa o Catch).

Camada 1 (Front-End) O Escudo do Navegador (cliente.js)

O evento DOMContentLoaded garante que o código apenas executa depois da página estar pronta. Aqui utilizamos o FormData para recolher todos os dados numa só linha e adicionamos o try/catch para proteger a página se a internet falhar.

cliente.js
document.addEventListener('DOMContentLoaded', () => {
    const form = document.getElementById('dcat-form');
    const output = document.getElementById('server-output');

    form.addEventListener('submit', async (event) => {
        event.preventDefault();
        
        // 1. A Magia do FormData (Extrai tudo de uma vez!)
        const formData = new FormData(form);
        // Transforma o FormData num Objeto JS normal { remetente: '...', feitico: '...' }
        const dados = Object.fromEntries(formData.entries());

        output.classList.remove('hidden');
        output.innerHTML = '<div class="spell-loader"></div> A contactar o Ministério...';

        // 2. A Rede de Segurança: TRY / CATCH
        try {
            const resposta = await fetch('http://localhost:3000/api/magia', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify(dados)
            });

            // Extrai o JSON de resposta (Seja de sucesso ou de erro)
            const dadosServidor = await resposta.json();

            // 3. Validação do Código de Status HTTP
            // .ok é true apenas se o status for 200-299
            if (!resposta.ok) {
                // Se deu erro 400, 403, etc... atiramos o erro para o Catch!
                throw new Error(`[STATUS ${resposta.status}] ${dadosServidor.erro}`);
            }

            // Se chegou aqui, é porque foi Status 200 OK!
            output.innerHTML = `<strong style="color:#10b981">Sucesso:</strong> ${dadosServidor.mensagem}`;

        } catch (erro) {
            // 4. Cai aqui se a internet falhar OU se o servidor devolver Status de Erro
            output.innerHTML = `<strong style="color:#ef4444">Proteção Ativada:</strong> ${erro.message}`;
        }
    });
});

Camada 2 (Back-End) O Juiz Implacável (servidor.js)

No servidor, nós nunca confiamos no cliente. Verificamos se os campos estão vazios (evitando lixo na base de dados) e devolvemos os Status Codes oficiais da web usando res.status(numero).json(...).

servidor.js (Node.js/Express)
const express = require('express');
const cors = require('cors'); 
const app = express();

app.use(cors()); 
app.use(express.json());

app.post('/api/magia', (req, res) => {
    
    // Desestruturação: Retira as variáveis diretamente do body
    const { remetente, feitico } = req.body;

    // --- REGRA 1: VALIDAÇÃO DUPLA (Campos Vazios) ---
    if (!remetente || !feitico) {
        // Devolve Status 400 (Bad Request)
        return res.status(400).json({ erro: 'Campos incompletos. Preencha tudo!' });
    }

    // --- REGRA 2: REGRAS DE NEGÓCIO (Magia Negra) ---
    if (remetente.toLowerCase().includes('voldemort')) {
        // Devolve Status 403 (Forbidden / Proibido)
        return res.status(403).json({ erro: 'Acesso proibido a feitiçaria das trevas.' });
    }

    if (feitico.toLowerCase() === 'avada kedavra') {
        // Devolve Status 500 (Simulando uma falha crítica no Servidor)
        return res.status(500).json({ erro: 'Dano crítico no núcleo do servidor!' });
    }

    // --- SUCESSO: O feitiço passou nas defesas! ---
    // O Express envia o Status 200 (OK) por defeito se não dissermos nada.
    res.status(200).json({ mensagem: `O feitiço ${feitico} foi registado com segurança.` });
});

app.listen(3000, () => console.log('Ministério da Magia online na porta 3000'));
