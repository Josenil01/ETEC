Aula 27: As Regras da Arena

Bem-vindo aos Jogos. Aprenda a separar a Interface do utilizador da Lógica de Negócio (Domínio) criando classes independentes que ditam as leis da física da sua aplicação.

1. A Ilusão do Controlo

Muitos programadores iniciantes cometem um erro fatal: colocam as regras do jogo dentro dos botões do HTML. Por exemplo, criam um if (vida < 0) { ... } dentro do addEventListener('click'). Isto chama-se acoplamento forte. Se amanhã o jogo for controlado por voz ou por um temporizador, eles terão de reescrever a regra da vida toda de novo.

Na verdadeira Arquitetura de Software, introduzimos o conceito de Domínio do Negócio (Domain). O Domínio é um conjunto de Ficheiros (Classes) que ignoram completamente a existência da Internet, do HTML, ou de Botões. Eles são a "Alma" da aplicação. Uma classe Jogador sabe como sofrer dano e morrer sozinha. O Front-End (Interface) é apenas um peão cego que desenha o que a classe do Domínio lhe ordena.

2. Mapeando o Universo

Entidade: Jogador 067

A unidade básica do nosso jogo. Ele é responsável exclusivo por gerir a sua própria vida e o seu estado vital. Não podemos alterar a vida dele diretamente do exterior (Encapsulamento).

  • Atributos: #vida, numero, status
  • Métodos: sofrerDano(), estaVivo()
O Gestor: Arena

A Entidade que agrega e gere as outras. O Jogador não sabe quem são os outros jogadores. É a Arena que guarda a lista de todos os tributos e verifica quando o jogo acaba.

  • Atributos: #tributos, #jogoAtivo
  • Métodos: adicionarTributo(), aplicarCastigo()

A Arquitetura Oculta

Estrutura do Projeto (Regra dos Módulos)

Tal como na Aula 26, usaremos a separação em ficheiros. Não se esqueça do Live Server! A arquitetura de Pastas (Domínio vs Controller) é o que separa um amador de um profissional.

├── 📄 index.html            (A Interface)
├── 📁 domain
│   ├── 📄 Jogador.js       (A Entidade que sofre a regra)
│   └── 📄 Arena.js         (O Controlador de regras globais)
└── 📄 main.js               (O Script Visual que ouve o HTML)

/domain/Jogador.js 3. A Entidade Blindada

O Jogador não sabe o que é uma div ou um botão. Ele é matemática pura. Repare no encapsulamento estrito da #vida.

Jogador.js
export class Jogador {
    #vida;
    #status; // "VIVO" ou "ELIMINADO"

    constructor(numeroIdentificacao) {
        this.numero = numeroIdentificacao;
        this.#vida = 100;
        this.#status = "VIVO";
    }

    // Getters (Permitem ler, mas nunca escrever diretamente)
    getVida() { return this.#vida; }
    getStatus() { return this.#status; }

    // A Regra de Negócio Física
    sofrerDano(quantidade) {
        if (this.#status === "ELIMINADO") return; // Não pode sofrer dano se já foi eliminado

        this.#vida -= quantidade;

        // A vida nunca desce abaixo de Zero na física deste universo
        if (this.#vida <= 0) {
            this.#vida = 0;
            this.#status = "ELIMINADO";
        }
    }
}

/domain/Arena.js 4. O Gestor Supremo

A Arena recebe "Jogadores" no seu construtor. A sua regra principal é aplicar castigos e verificar se há um vencedor (quando sobra apenas 1 jogador vivo).

Arena.js
export class Arena {
    #tributos;
    #vencedor;

    constructor(arrayDeJogadores) {
        this.#tributos = arrayDeJogadores;
        this.#vencedor = null;
    }

    getTributos() { return this.#tributos; }
    getVencedor() { return this.#vencedor; }

    // Método que age sobre os outros objetos!
    aplicarCastigo(indexJogador, dano) {
        if (this.#vencedor) return "O jogo já acabou.";

        const alvo = this.#tributos[indexJogador];
        alvo.sofrerDano(dano);
        
        this.#verificarFimDeJogo(); // Verifica regras a cada hit
        
        return `O Jogador ${alvo.numero} sofreu ${dano} de dano.`;
    }

    // Método Privado (Só a Arena pode chamar isto internamente)
    #verificarFimDeJogo() {
        // Filtra o array devolvendo apenas os que têm status "VIVO"
        const vivos = this.#tributos.filter(j => j.getStatus() === "VIVO");
        
        if (vivos.length === 1) {
            this.#vencedor = vivos[0];
        }
    }
}

Controller 5. A Camada Visual (main.js)

Este ficheiro é o maestro do DOM. Ele junta as classes e os botões do index.html. Quando clica em "Castigar", o main.js pede à Arena para fazer as contas e, de seguida, desenha (renderiza) o resultado atualizado na página com o style.css apropriado.

main.js
import { Jogador } from './domain/Jogador.js';
import { Arena } from './domain/Arena.js';

// 1. Instanciamos as Regras de Negócio (O Jogo Invisível)
const p1 = new Jogador('456');
const p2 = new Jogador('067');
const p3 = new Jogador('218');

const jogos = new Arena([p1, p2, p3]);

// 2. A Função que converte as classes para HTML
function renderizarInterface() {
    const grid = document.getElementById('jogadores-grid');
    grid.innerHTML = '';

    // Desenha cada jogador
    jogos.getTributos().forEach(jog => {
        const isMorto = jog.getStatus() === 'ELIMINADO';
        
        grid.innerHTML += `
            <div class="jogador-card ${isMorto ? 'eliminado' : ''}">
                <div class="numero-jogador">${jog.numero}</div>
                <div class="barra-vida-bg">
                    <div class="barra-vida-fill ${jog.getVida() < 50 ? 'low' : ''}" style="width: ${jog.getVida()}%"></div>
                </div>
                <span class="text-xs font-bold">VIDA: ${jog.getVida()}</span>
            </div>
        `;
    });

    // Verifica Vencedor
    const vencedor = jogos.getVencedor();
    if (vencedor) {
        document.getElementById('vencedor-display').innerText = `👑 VENCEDOR: ${vencedor.numero}`;
        // Desativa todos os botões
        document.querySelectorAll('.attack-btn').forEach(btn => btn.disabled = true);
    }
}

// 3. Ação ativada pelo HTML: onclick="castigar(index)"
window.castigar = function(index) {
    const resultadoAcao = jogos.aplicarCastigo(index, 35); // Aplica 35 de dano
    document.getElementById('arena-log').innerText = `> ${resultadoAcao}`;
    
    // Mandamos o interface desenhar o novo estado físico do jogo!
    renderizarInterface(); 
};

// Desenha a tela inicial logo ao carregar
renderizarInterface();

4. Provações do Front Man (Prática)

Nível: Guarda Círculo (Fácil)

01

Kit de Primeiros Socorros: Vá à classe Jogador e adicione um novo método chamado curar(quantidade). A regra de negócio é: Aumente a vida pelo valor recebido, mas adicione um if para garantir que a vida nunca ultrapassa o limite de 100. Depois teste no main.js.

Nível: Guarda Triângulo (Médio)

02

Uma Nova Entidade (A Arma): A regra é separar responsabilidades. Crie um novo ficheiro domain/Arma.js. Escreva uma classe Arma com atributos nome e poderDeDano. Instancie-a no main.js e use o arma.poderDeDano como o valor que passa para o método aplicarCastigo() da Arena.

Nível: O Front Man (Desafio)

03

A Regra da Noite (Lógica Global):

1. Na classe Arena, crie um novo atributo privado #ehNoite = false; e um método alternarDiaNoite().
2. Na sua interface do HTML, adicione um botão "Desligar Luzes".
3. Modifique o método aplicarCastigo(index, dano) da Arena: se #ehNoite for true, o dano recebido pelo jogador deve ser o dobro (dano * 2).

Com isto, você alterou a gravidade da aplicação sem tocar na classe do Jogador, provando que o Domínio está perfeitamente arquitetado!
