Aula 11: Formulários e Validação

Aprenda a intercetar formulários, impedir o recarregamento da página e validar os dados dos utilizadores antes de autorizar o acesso.

1. A Iniciativa Vingadores

Regra número um do desenvolvimento web: Nunca confie nos dados introduzidos pelo utilizador.

A S.H.I.E.L.D. não pode permitir que agentes se registem sem um nome de código ou com um nível de poder inválido. Como programadores, o nosso trabalho é usar o JavaScript para intercetar o botão "Enviar", verificar se os dados inseridos nas caixas de texto estão corretos e, só então, dar luz verde. Isso chama-se Validação de Formulários no Front-End.

2. Ferramentas do Agente

Interceptação de Formulário
evento 'submit'

Em vez de escutar o 'click' do botão, escutamos o 'submit' do formulário inteiro. Isso cobre tanto cliques no botão quanto apertar "Enter".

event.preventDefault()

Crucial! Bloqueia o comportamento padrão do HTML (que é atualizar a página imediatamente ao enviar um formulário). Isso permite que o JS valide os dados antes.

Leitura de Dados
input.value

Propriedade que captura exatamente o que está escrito dentro da caixa de texto no momento em que o formulário foi submetido.

input.value.trim()

Método que remove espaços em branco acidentais no início e no final do texto (ex: `" Tony "` vira `"Tony"`).

3. Exemplo Prático: Passo a Passo

Experimente tentar enviar o formulário vazio, ou com uma senha curta. Veja como o JavaScript bloqueia a ação e fornece feedback visual.

Terminal S.H.I.E.L.D. (Preview)

⚠️ O nome não pode estar vazio.

⚠️ O código deve ter pelo menos 6 caracteres.

✅ Acesso Concedido, Agente!

Passo a Passo da Implementação

Passo 1 A Estrutura do Formulário (HTML)

Veja a estrutura base do <form>. Deixamos as mensagens de erro prontas, mas ocultas com a classe hidden do Tailwind. O JS irá remover essa classe quando encontrar erros.

Acesso Restrito.

Apenas digitação autorizada. Cópia não permitida.

index.html (Estrutura do Formulário)
<!-- O ID no form é crucial para interceptarmos o evento 'submit' -->
<form id="form-recruta">
    
    <div>
        <label>Nome de Código</label>
        <input type="text" id="input-nome">
        <!-- Mensagem de erro inicialmente escondida (hidden) -->
        <p id="erro-nome" class="hidden error-text">⚠️ Nome vazio.</p>
    </div>

    <div>
        <label>Código de Acesso</label>
        <input type="password" id="input-codigo">
        <p id="erro-codigo" class="hidden error-text">⚠️ Mín. 6 caracteres.</p>
    </div>

    <!-- O botão type="submit" dispara o envio do formulário -->
    <button type="submit">Autorizar Acesso</button>

    <div id="msg-sucesso" class="hidden success-box">
        ✅ Acesso Concedido, Agente!
    </div>

</form>

Passo 3 Seleção e preventDefault (JS)

Identificamos o formulário e usamos o event.preventDefault() para impedir que a página recarregue ao clicar em enviar.

seguranca.js (Parte 1)
// 1. Selecionamos os elementos
const formulario = document.getElementById('form-recruta');
const inputNome = document.getElementById('input-nome');
const inputCodigo = document.getElementById('input-codigo');

// 2. Escutamos o evento 'submit' DO FORMULÁRIO (não do botão)
formulario.addEventListener('submit', (event) => {
    
    // OBRIGATÓRIO: Impede o recarregamento imediato da página
    event.preventDefault();
    
    // O restante do código de validação entrará aqui!
});

Passo 4 Lógica de Validação com IFs

Dentro do evento submit, lemos a propriedade .value. Se estiver errado, mostramos o erro; se estiver tudo certo, mostramos o sucesso.

seguranca.js (Parte 2 - Dentro do EventListener)
    // .trim() limpa espaços extras digitados pelo usuário
    const nomeDigitado = inputNome.value.trim();
    const codigoDigitado = inputCodigo.value.trim();

    let temErro = false;

    // --- VALIDAÇÃO DO NOME ---
    if (nomeDigitado === "") {
        document.getElementById('erro-nome').classList.remove('hidden');
        inputNome.classList.add('input-error');
        temErro = true;
    } else {
        document.getElementById('erro-nome').classList.add('hidden');
        inputNome.classList.remove('input-error');
    }

    // --- VALIDAÇÃO DO CÓDIGO (SENHA) ---
    if (codigoDigitado.length < 6) {
        document.getElementById('erro-codigo').classList.remove('hidden');
        inputCodigo.classList.add('input-error');
        temErro = true;
    } else {
        document.getElementById('erro-codigo').classList.add('hidden');
        inputCodigo.classList.remove('input-error');
    }

    // --- DECISÃO FINAL ---
    if (temErro === false) {
        // Se passou nas validações, mostra sucesso!
        document.getElementById('msg-sucesso').classList.remove('hidden');
        
        // Limpa os campos
        inputNome.value = "";
        inputCodigo.value = "";
    } else {
        document.getElementById('msg-sucesso').classList.add('hidden');
    }

4. Avaliação de Campo (Prática)

Nível: Agente Novato (Fácil)

01

Parar a Ameaça: Crie um <form id="login"> simples com um botão. No JS, adicione um addEventListener('submit'). Dentro da função, coloque apenas event.preventDefault() e um alert('O formulário foi bloqueado!'). Clique no botão e garanta que a página não recarrega.

02

Leitura Básica: No mesmo formulário, adicione um <input type="text" id="heroi">. No JS do submit, use um console.log() para imprimir na tela exatamente o .value que foi digitado no input.

Nível: Vingador (Médio)

03

Validador de Idade: Crie um formulário com um input de Idade (type="number"). Ao fazer submit, leia o .value. Escreva um bloco if/else: Se a idade for menor que 18, altere o innerText de um parágrafo vazio para "Acesso Negado: Muito jovem". Se for maior ou igual, mostre "Acesso Permitido".

Nível: Diretor da S.H.I.E.L.D. (Desafio)

04

Identificação de Senhas Iguais:

1. Crie um formulário de "Nova Senha" com dois campos: id="senha1" e id="senha2" (Confirmar Senha).
2. Intercepte o submit do formulário.
3. Extraia o value de ambos os campos.
4. Crie a regra: Se senha1 for diferente (!==) de senha2, use o JavaScript para adicionar uma classe com borda vermelha (ou usar .style.border = "2px solid red") em ambos os campos, alertando o utilizador que as senhas não coincidem!
