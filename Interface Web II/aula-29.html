Aula 29: A Base de Operações

O fim do código esparguete. Aprenda a usar o Padrão Router & Controller para dividir o seu Servidor Express num ecossistema perfeito e altamente escalável.

1. O Monstro Monolítico

Na Aula 28 (Os Caçadores de Relíquias), nós fizemos um trabalho brilhante: retirámos o código SQL (PostgreSQL) do nosso servidor.js e escondemo-lo no ReliquiaRepository.js. O nosso servidor parecia limpo.

Mas há um problema: Imagine que o nosso Museu cresce. Agora precisamos de gerir rotas para Relíquias (GET, POST, PUT, DELETE), Exploradores (GET, POST), Expedições e Login. Se colocarmos todos esses `app.get` e `app.post` num único ficheiro servidor.js, ele rapidamente terá milhares de linhas de código. Torna-se num monstro incontrolável (O Monólito).

A solução definitiva adotada pela indústria é a Arquitetura API RESTful (Padrão Router & Controller). O ficheiro principal passará a ser apenas um "Gestor" que delega o trabalho para pequenos módulos especializados.

2. A Hierarquia do Museu (MVC)

O Diretor (servidor.js)

Apenas abre as portas do museu (porta 3000), liga a segurança (CORS) e delega os visitantes para o Guia correto.

app.use('/api', rotas)
O Guia (Routes)

Lê o mapa (A URL). Se o visitante for para /api/reliquias com método POST, ele encaminha-o para a sala do Curador responsável pelas relíquias.

router.post('/', func)
O Curador (Controller)

É aqui que os blocos `try/catch` vivem! O Controller recebe o pedido (req), aciona o Repositório (Base de Dados), e envia a resposta final (res.json()).

class ReliquiaController

/controllers 1. Criando o Curador (ReliquiaController.js)

Nós pegamos em todo aquele bloco try/catch gigante que estava na Aula 28 e colocamo-lo dentro de métodos de uma nova classe chamada ReliquiaController. Instanciamos o Repositório no constructor e usamos Arrow Functions nos métodos para não perdermos a referência ao this. O Controller é o único que sabe usar os comandos req (ler dados do cliente) e res (enviar resposta em JSON). Não se esqueça da linha module.exports no final do ficheiro; é ela que dá permissão para que as rotas consigam importar este módulo!

ReliquiaController.js
// Importamos as regras de negócio e o banco de dados
const Reliquia = require('../domain/Reliquia.js');
const ReliquiaRepository = require('../repositories/ReliquiaRepository.js');

class ReliquiaController {
    
    constructor() {
        // Instancia o repositório como propriedade da classe
        this.repo = new ReliquiaRepository();
    }

    // Método para gerir o pedido GET (Listar) usando Arrow Function
    listar = async (req, res) => {
        try {
            const catalogo = await this.repo.listarTodas();
            res.json(catalogo);
        } catch (erro) {
            console.error(erro);
            res.status(500).json({ erro: 'Falha interna no servidor.' });
        }
    }

    // Método para gerir o pedido POST (Criar) usando Arrow Function
    criar = async (req, res) => {
        try {
            const { nome, epoca } = req.body;
            const novoArtefato = new Reliquia(nome, epoca);
            
            novoArtefato.validar(); // Domínio rico a proteger!

            const artefatoGuardado = await this.repo.salvar(novoArtefato);
            res.status(201).json(artefatoGuardado);

        } catch (erro) {
            res.status(400).json({ erro: erro.message });
        }
    }
}

module.exports = ReliquiaController;

/routes 2. O Guia do Museu (reliquiasRoutes.js)

Este ficheiro é um mini-servidor especializado apenas em gerir URLs ligadas a relíquias. Usamos o express.Router() para ligar a URL de destino diretamente ao Método do Controller que criámos acima. Não se esqueça da linha module.exports no final do ficheiro; é ela que dá permissão para que o ficheiro servidor.js consiga importar este módulo!

reliquiasRoutes.js
const express = require('express');
const router = express.Router();

// 1. Importamos e instanciamos o nosso Controller (Curador)
const ReliquiaController = require('../controllers/ReliquiaController.js');
const controller = new ReliquiaController();

// 2. Mapeamos a URL para a função correta
// Repare que a URL base ("/api/reliquias") será definida no servidor.js! 
// Aqui, "/" significa exatamente a raiz dessa URL base.
router.get('/', controller.listar);
router.post('/', controller.criar);

// Exemplo futuro de deleção (O parâmetro de ID fica aqui na Rota)
// router.delete('/:id', controller.deletar);

module.exports = router;

/ 3. O Diretor Supremo (servidor.js limpo)

Olhe para a beleza deste ficheiro principal. Ele não sabe nada de base de dados, não sabe nada de SQL, e não sabe fazer try/catches de regras de negócio. O seu único trabalho é iniciar a aplicação, aplicar os seguranças (CORS/JSON) e ligar o Router a uma morada base na web.

servidor.js
const express = require('express');
const cors = require('cors');

// 1. Importamos os Guias (Routes) do nosso museu
const reliquiasRoutes = require('./routes/reliquiasRoutes.js');
// const exploradoresRoutes = require('./routes/exploradoresRoutes.js');

const app = express();
app.use(cors());
app.use(express.json());

// 2. A DELEGAÇÃO:
// Dizemos ao servidor: "Tudo o que começar com /api/reliquias, envia para o reliquiasRoutes resolver!"
app.use('/api/reliquias', reliquiasRoutes);

// Imagine o quão fácil é adicionar novas alas ao museu agora:
// app.use('/api/exploradores', exploradoresRoutes);

app.listen(3000, () => console.log('Museu Central online na porta 3000'));

A Grande Vitória (Front-End Intacto)

Reparou que não mexemos numa única linha do ficheiro cliente.js do nosso Front-End? Como mantivemos a URL /api/reliquias intacta no Servidor, o navegador nem sequer sabe que o Back-End foi desconstruído e organizado noutras pastas. Este é o verdadeiro poder e a beleza do Desacoplamento de APIs!

02

A Operação de Destruição (Delete): Na Aula 28 (Desafio), pedimos para criar um DELETE. Refatore essa lógica! Vá ao seu ReliquiaController.js e adicione um novo método assíncrono usando a sintaxe de Arrow Function: apagar = async (req, res) => { ... }. Extraia o ID via req.params.id. Use um bloco try/catch para chamar o Repositório e responder ao cliente.

Nível: Arquiteto Chefe (Desafio)

03

O Mistério das Arrow Functions:

Reparou que no Passo 1 definimos os nossos métodos do Controller como listar = async (req, res) => { ... } em vez do tradicional async listar(req, res) { ... }?

Se tentarmos usar a sintaxe tradicional e passarmos esse método para o Router do Express (router.get('/', controller.listar)), vamos receber um erro no terminal a dizer que this.repo is undefined quando a rota for chamada!

A Missão: Pesquise na internet sobre a "Perda do contexto do this no Express" ou "Perda de contexto de métodos em JavaScript". Descubra como e porquê as Arrow Functions resolvem este clássico problema de arquitetura no NodeJS de forma tão elegante!
