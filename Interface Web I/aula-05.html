Aula 05: A Ponte de Dados

Estrutura de Formulários e Entradas Básicas

O Questionário de Papel

Criar um formulário web é exatamente como desenhar um Questionário de Inscrição em papel.

O Envelope (<form>)

O formulário é o envelope que guarda as respostas. Sem ele, os dados ficam soltos e o servidor não sabe para onde devem ser enviados ou como devem ser processados.

Etiqueta e Campo

A label é a pergunta escrita no papel ("Qual o seu nome?"). O input é o espaço em branco onde o utilizador escreve a resposta. Estão ligados por um fio invisível (o ID).

1. Componentes da Interatividade

Action e Method

O action define o destino (URL) dos dados. O method define como viajam: GET (visível na URL) ou POST (invisível e seguro).

O Atributo 'for'

Sempre use <label for="id-do-campo">. Isso permite que, ao clicar no texto da etiqueta, o cursor foque automaticamente no campo de texto.

Tipos de Input

text (geral), email (valida o @), password (esconde caracteres) e number (aceita apenas algarismos).

Placeholders e Required

placeholder é a dica cinzenta dentro do campo. required impede o envio se o utilizador deixar o campo vazio.

2. Prática: O Formulário de Registo

Analise o código abaixo de um formulário de subscrição moderno. Cada linha possui um comentário didático:

<!-- A tag 'form' envolve todos os campos. O '#' no action significa 'mesma página' -->
<form action="#" method="POST">

    <!-- Divisão para organizar o grupo do Nome -->
    <div>
        <!-- O 'for' liga esta etiqueta ao input com id='user-name' -->
        <label for="user-name">Nome Completo:</label><br>
        <!-- 'name' é o nome da variável que o servidor vai receber -->
        <input type="text" id="user-name" name="nome" placeholder="Ex: João Silva" required>
    </div>

    <br>

    <!-- Divisão para organizar o grupo do Email -->
    <div>
        <label for="user-email">Endereço de Correio Eletrónico:</label><br>
        <!-- 'type=email' garante que o utilizador digite um formato válido -->
        <input type="email" id="user-email" name="email" placeholder="exemplo@dominio.com">
    </div>

    <br>

    <!-- Divisão para organizar o grupo da Palavra-passe -->
    <div>
        <label for="user-pass">Palavra-passe:</label><br>
        <!-- 'type=password' mascara os caracteres com pontos por segurança -->
        <input type="password" id="user-pass" name="senha" minlength="8" required>
    </div>

    <br>

    <!-- Botão que dispara o envio do formulário -->
    <button type="submit">Criar Conta Agora</button>

    <!-- Botão que limpa todos os campos para o estado original -->
    <button type="reset">Limpar Dados</button>

</form>

3. Atividade: O Arquiteto de Interações

Modifique o código base para tornar o formulário mais robusto:

1

Novo Campo: Adicione um campo para "Telemóvel" usando o type="tel". Não se esqueça de criar a etiqueta correspondente.

2

Validação Extra: No campo de Palavra-passe, adicione o atributo maxlength="20" para limitar o tamanho da senha.

3

Campo Numérico: Insira uma pergunta "Quantos anos tem?" usando o type="number" com os atributos min="13" e max="100".

4

Dicas Visuais: Altere os placeholder para que sejam exemplos reais de dados que o utilizador deve inserir.

5

Envio Seguro: No cabeçalho da tag <form>, altere o method para GET e tente submeter o formulário. O que aparece na barra de endereços do seu navegador? (Depois volte para POST).
